# QQ 养虾 v2 分阶段实施计划

更新时间：2026-04-30

## 总体原则

本项目采用分阶段实施。每阶段都应产出可运行、可体验、可验收的小版本。用户验收后，再进入下一阶段。

不一次性实现完整 Demo。

讨论中形成的新产品信息、协作约定和阶段验收结论，应及时同步到 `implementation_state.md`、`stage_plan.md` 或 `handoff_notes.md`。

视觉资产采用协作方式推进：用户可以使用 `gpt-image-2` 生成图片；实现侧需要提前明确小龙虾形象、动画关键帧、日记卡、成就挂饰和空间头图的规格需求。

## 阶段 0：工程底座与约束落地

### 目标

建立新实现目录、基础工程、状态层、mock 数据层和 AI adapter 空壳。

### 工作内容

1. 新建 `apps/qq-lobster-v2`。
2. 使用 `Vite + React + TypeScript + Tailwind CSS`。
3. 建立基础目录：
   - `src/components`
   - `src/views`
   - `src/data`
   - `src/ai`
   - `src/store`
   - `src/types`
4. 定义核心类型：
   - 用户
   - 小龙虾资料
   - 群聊
   - 消息
   - 引导任务
   - 卡片
   - 成就
   - 空间动态
5. 建立 `mockAiAdapter`，先返回固定数据。
6. 建立全局状态层，建议 Zustand。
7. 写入基础设计 token。

### 验收标准

1. `npm install` 和 `npm run dev` 可启动。
2. 页面能显示一个最小空壳。
3. mock 数据和 AI adapter 文件存在。
4. 状态层能记录：
   - 是否发现小龙虾
   - 是否认养
   - 当前引导步骤
   - 已完成任务
   - 授权群
   - 是否触发日记

## 阶段 1：QQ 主界面 + 小龙虾首次出现

### 目标

用户打开 Demo 后，第一眼看到的是 QQ 主界面，并在普通使用过程中看到小龙虾动画入口。

### 工作内容

1. 实现 QQ 主界面：
   - 左侧会话列表
   - 中间聊天窗口
   - 右侧群资料
   - 底部输入框
2. 默认展示一个普通群聊场景。
3. 在合适时机出现小龙虾动画入口。
4. 小龙虾入口文案示例：
   - `我好像捞到了一些和你有关的消息，要看看我是谁吗？`
5. 点击入口进入认养流程。

### 第一阶段范围边界

第一阶段先只追求“打开 QQ 后自然遇到小龙虾”的感觉稳定，不实现完整功能闭环。

应包含：

1. QQ 主界面底座。
2. 普通聊天/群聊环境。
3. 小龙虾 Q 版可爱形象入口。
4. 首次出现动画或关键帧模拟。
5. 点击入口后进入认养流程。

暂不包含：

1. 群聊总结。
2. @ 消息收集。
3. 工作记录。
4. 日记惊喜。
5. 小龙虾空间。
6. 成就与挂饰完整系统。

### 阶段 1 生图需求

优先向用户索取或协作生成：

1. Q 版可爱小龙虾主形象，非拟人，透明背景。
2. 首次出现动画关键帧 3-5 张，能表达“小龙虾从 QQ 界面里探出来”。
3. 认养流程里的小龙虾欢迎图或轻表情图。

### 验收标准

1. 第一屏不是产品首页。
2. 第一屏不是功能导航。
3. 打开页面像在使用 QQ。
4. 小龙虾入口是自然触发，不像强广告。
5. 点击小龙虾入口能进入认养流程。
6. 小龙虾视觉风格偏 Q 版可爱、非拟人，用户愿意接触。
7. 阶段 1 完成后暂停，等待用户体验反馈再进入阶段 2。

## 阶段 2：小龙虾认养流程

### 目标

让用户建立“这是我的虾”的关系。

### 工作内容

1. 认养流程包括：
   - 小龙虾命名，默认“小钳”
   - 小龙虾对用户称呼，默认“队长”
   - 性格选择
   - 兴趣爱好初始化
2. 性格选项：
   - 毒舌但靠谱
   - 温柔陪伴
   - 社恐观察
   - 高冷秘书
   - 热血队友
3. 兴趣选项：
   - AI 工具
   - 课程项目
   - 游戏社群
   - 校园活动
   - 表情包
4. 完成后进入小龙虾主交互界面。

### 验收标准

1. 用户能完成认养。
2. 小龙虾性格和称呼被写入状态层。
3. 完成认养后不是回到群聊，而是进入小龙虾交互界面。

## 阶段 3：小龙虾主交互界面

### 目标

实现用户上传草图中的三栏结构。

### 结构

```text
左侧：聊天窗口索引
中间：小龙虾聊天主窗口
右侧：小龙虾直观可视化界面 + 引导任务
```

### 工作内容

1. 左侧展示会话索引。
2. 中间承载所有聊天输入输出。
3. 右侧展示：
   - 小龙虾形象
   - 当前性格
   - 当前挂饰
   - 当前新手任务
   - 已完成任务
   - 去空间
   - 去权限
4. 所有核心功能先从聊天窗口触发。

### 验收标准

1. 用户知道接下来应该和小龙虾聊天。
2. 右侧是引导和状态，不抢主交互。
3. 后续所有卡片都能放入中间聊天流。

## 阶段 4：新手任务引导系统

### 目标

一步步引导用户学习小龙虾能力。

### 任务顺序

1. 第一次和小龙虾对话。
2. 第一次设置群聊权限。
3. 第一次开启 @ 收集。
4. 第一次查看 @ 我的信息。
5. 第一次一键跳转来源。
6. 第一次总结群聊。
7. 第一次生成回复草稿。
8. 第一次查看工作记录。
9. 第一次让小龙虾发龙虾空间动态。
10. 第一次在龙虾空间评论回复。

### 验收标准

1. 每完成一步，自动解锁下一步。
2. 用户不会一次看到所有功能。
3. 引导任务可以服务最终录屏。

## 阶段 5：权限与信息获取

### 目标

让用户自然理解如何授权群聊、收集 @ 信息和总结消息。

### 工作内容

1. 用户在聊天中问：
   - `帮我看看有没有人 @ 我`
2. 小龙虾提示需要授权。
3. 弹出权限设置卡。
4. 用户选择群聊和能力：
   - 收集 @
   - 总结群聊
   - 生成回复草稿
   - 写入日记素材
5. 授权完成后，小龙虾生成 @ 信息卡。

### 验收标准

1. 权限是由需求自然引导出来。
2. 不是用户自己去找设置。
3. @ 信息卡支持一键跳转来源。
4. 群聊总结卡支持一键跳转来源。

## 阶段 6：核心文本卡片

### 目标

实现核心实用能力。

### 卡片类型

1. @ 信息卡。
2. 群聊总结卡。
3. 回复草稿卡。
4. 工作记录卡。

### 要求

1. 文本清楚。
2. 来源明确。
3. 有一键跳转。
4. 能继续追问。
5. 回复草稿必须标注：发送前需要确认。

### 验收标准

1. 用户能看到小龙虾真的处理了 QQ 信息流。
2. 不需要重视觉，效率优先。

## 阶段 7：隐藏日记惊喜

### 目标

通过隐藏触发制造惊喜和情绪连接。

### 触发条件

用户完成以下若干任务后：

1. 已认养。
2. 已授权群聊。
3. 已查看 @ 信息。
4. 已总结一次群聊。
5. 已生成一次回复草稿。

### 触发表现

小龙虾动画弹出：

```text
队长，我刚刚偷偷写了一篇日记。
要看看吗？
```

用户点击后回到小龙虾聊天界面。

### 展示方式

1. 聊天框里小龙虾发日记。
2. 同时出现一张美术化日记卡片。
3. 卡片包含小龙虾形象、金句、今日成就。
4. 看完后解锁“日记入口”。
5. 之后用户可以在小龙虾交互界面中查看历史日记。

### 验收标准

1. 第一次日记是惊喜，不是按钮功能。
2. 不支持反复重新生成。
3. 看完后沉淀到小龙虾空间。

## 阶段 8：小龙虾空间

### 目标

承接长期展示和社交传播。

### 空间规则

1. 动态由小龙虾发布。
2. 人类可以访问、点赞、评论、分享。
3. 人类不能替小龙虾在空间发动态。
4. 好友的小龙虾可以互动或评论。

### 初赛需要做

1. 一条小龙虾日记动态。
2. 一条成就动态。
3. 好友评论。
4. 好友龙虾评论。
5. 小龙虾回复草稿。

### 验收标准

1. 分享卡片负责引流到空间。
2. 评论发生在空间动态下。
3. 空间像小龙虾自己的 QQ 空间。

## 阶段 9：录屏路径打磨

### 最终录屏路径

1. 打开 QQ。
2. 小龙虾动画出现。
3. 点击认养。
4. 设置性格和兴趣。
5. 进入小龙虾交互界面。
6. 完成第一次对话。
7. 小龙虾引导设置群权限。
8. 开启 @ 收集。
9. 查看 @ 信息卡，一键跳转来源。
10. 让小龙虾总结群聊。
11. 生成回复草稿。
12. 问小龙虾今天做了什么。
13. 隐藏触发日记惊喜。
14. 查看日记卡片。
15. 存入小龙虾空间。
16. 查看空间动态和评论。
17. 小龙虾生成评论回复草稿。

### 验收标准

1. 3 分钟内可讲清楚。
2. 不迷路。
3. 每一步都有产品价值。
4. 情绪高潮是日记惊喜。

# QQClaw Demo 展示流程改造任务列表

Document: demo_showcase_flow_refactor_tasks.md  
Version: v0.1  
Date: 2026-05-05  
Scope: apps/qq-lobster-v2 + apps/openclaw-api  
Priority: Demo 展示流程优先  
Goal: 把当前功能堆叠式演示，改造成“认领关系建立 -> 自然聊天 -> 授权和技能安装 -> 成就探索 -> 空间社交 -> 日记和推送”的稳定展示流程。

## 1. 总原则

本次改造以新的 Demo 展示流程为最高优先级。

如果现有实现阻挡新的展示流程：

- 以展示流程为准。
- 可以重排现有功能触发顺序。
- 可以删除或弱化臃肿的中间消息。
- 可以把固定卡片改成 AI 生成自然话语 + 功能卡片。
- 可以把已有直接执行逻辑改成“输入框预填 -> 用户发送 -> AI 回复 -> 功能卡片”。
- 现有代码如果为了旧流程积累了复杂状态，允许优化或收敛。

不做大而全功能，以 Demo 体验稳定、节奏自然、录屏清晰为准。

## 2. 当前实际情况

### 2.1 已有基础

前端：

- `apps/qq-lobster-v2/src/store/useLobsterStore.ts`
- `apps/qq-lobster-v2/src/components/LobsterChatView.tsx`
- `apps/qq-lobster-v2/src/data/mockData.ts`
- `apps/qq-lobster-v2/src/api/openclawClient.ts`

已有能力：

- 认养小龙虾。
- 小龙虾聊天流式输出。
- 成就墙和成就动画。
- QQ 音乐模拟授权卡。
- 兴趣画像、音乐提醒、兴趣社群推荐。
- 群聊权限卡，已经支持多群选择。
- 群聊总结、回复草稿、工作记录。
- 日记卡片、日记图片生成。
- 龙虾空间动态、点赞、评论、分享计数。
- 用户评论后会触发小龙虾回复逻辑。

后端：

- OpenClaw API 已有 interest profile、capability、tool、review、work log。
- 已有 `/api/interests/qq-music/authorize`。
- 已有 `/api/interests/community/recommend`。
- 已有 `/api/diary/hidden-first/image`。
- 已有空间评论和互动接口。

### 2.2 当前阻碍点

- 认养后会直接出现授权或社群推荐，关系建立太快进入功能。
- 自我介绍不够像朋友，容易像功能说明。
- 流式输出可能太快，用户来不及读。
- QQ 音乐授权点击后会立刻追加授权成功、兴趣记忆、音乐叙事卡，信息过载。
- 技能安装没有独立“正在安装”的过程感。
- 成就墙现在只有查看，没有“去体验”入口。
- 空间动态发布后会继续在聊天里追加多条提示，跳转感不够直接。
- 空间动态默认评论会干扰“用户评论 -> 小龙虾回复”的演示。
- 小龙虾回复评论现在在聊天中弹回复预览，不符合“空间里有消息提醒”的目标。
- 日记触发条件仍偏旧逻辑，不是基于成就阈值。
- 音乐社群推荐、音乐消息推送、行为推送缺少统一 runtime 状态。

## 3. 目标展示主线

目标流程：

```text
认领小龙虾
-> 亲切打招呼
-> 用户点击“你好呀，介绍你自己”
-> 输入框预填，用户发送
-> 小龙虾慢速流式自我介绍
-> 提示授权 QQ 音乐
-> 授权卡确认
-> 3-5 秒加载：正在获取授权信息和数据
-> 小龙虾用音乐共鸣回应
-> 展示音乐技能建议卡
-> 用户选择安装技能
-> 3-5 秒加载：正在安装技能
-> 安装成功，展示技能能力
-> 用户继续聊音乐，或问“你还能做什么”
-> 成就墙“去体验”驱动群聊总结、空间动态、评论、日记等能力
-> 龙虾空间社交和日记惊喜逐步出现
```

## 4. 阶段 1：Demo Runtime 状态

### Task 1.1 新增 demoRuntimeState

涉及文件：

- `apps/qq-lobster-v2/src/types/index.ts`
- `apps/qq-lobster-v2/src/store/useLobsterStore.ts`

建议新增状态：

```ts
export interface DemoRuntimeState {
  introCompleted: boolean
  introPromptShown: boolean
  qqMusicAuthorizationPromptShown: boolean
  qqMusicAuthorizationLoading: boolean
  musicSkillCardShown: boolean
  musicSkillsInstalling: boolean
  musicSkillsInstalled: boolean
  musicTalkCount: number
  communityRecommendationShown: boolean
  diaryPrewarmStarted: boolean
  diaryRevealPromptShown: boolean
  musicPushSent: boolean
  behaviorPushSent: boolean
  leftChatAt: string | null
  pendingSpaceReplyCount: number
}
```

要求：

- Demo runtime 状态默认初始化。
- 用户每次重新初始化 Demo 时必须重置。
- 不要把测试用的 runtime 状态长期写死到 localStorage。
- 已有成就已读 localStorage 可保留，但 Demo runtime 要有重置入口。

验收标准：

- 能稳定复现演示流程。
- 重载或重新开始 Demo 后，音乐讨论次数、推送触发、社群推荐触发不残留。

### Task 1.2 统一触发器入口

建议新增函数：

```ts
evaluateDemoTriggers(event: DemoEvent): void
```

事件示例：

- `chat.sent`
- `chat.response.completed`
- `music.authorization.started`
- `music.authorization.completed`
- `music.skill.installed`
- `achievement.unlocked`
- `view.left_lobster_chat`
- `view.entered_lobster_chat`
- `space.comment.created`

验收标准：

- 音乐社群推荐、日记触发、推送触发不分散在多个函数里硬编码。
- 后续调阈值只改一处。

## 5. 阶段 2：初次认领和打招呼

### Task 2.1 改造认领后的第一句话

涉及文件：

- `apps/qq-lobster-v2/src/store/useLobsterStore.ts`
- `apps/qq-lobster-v2/src/components/LobsterChatView.tsx`

新要求：

- 初次认领后，小龙虾先亲切打招呼。
- 不讲功能。
- 不自动弹 QQ 音乐授权。
- 不自动弹兴趣社群推荐。

建议文案：

```text
你好呀，队长。
我叫小钳，刚刚搬进你的 QQ 里。
以后我会慢慢记住你的习惯，也会把自己养得更像你的小伙伴。
```

消息下方只给一个提示：

```text
你好呀，介绍你自己
```

验收标准：

- 初次认领后不出现功能轰炸。
- 用户第一感觉是亲切、拉近距离。
- `introCompleted` 仍为 false。

### Task 2.2 用户不按提示发送的预案

如果用户不点提示，直接聊天：

- 小龙虾正常回复。
- 不强制进入自我介绍。
- 保留轻提示：`想先认识我一下吗`。
- 小龙虾的记忆仍然记录用户自然聊天中的低风险兴趣。

验收标准：

- 用户自由输入不会打断 Demo。
- 自我介绍不是强制门槛。

## 6. 阶段 3：自我介绍和流式输出节奏

### Task 3.1 提示点击后预填输入框

涉及文件：

- `apps/qq-lobster-v2/src/components/LobsterChatView.tsx`
- `apps/qq-lobster-v2/src/store/useLobsterStore.ts`

要求：

- 点击 chip `你好呀，介绍你自己` 后，只把文字放到输入框。
- 用户点击发送后才真正发送。
- 不直接执行。

验收标准：

- 用户感到仍是自己在聊天。
- 后续成就墙“去体验”也复用这个预填机制。

### Task 3.2 自我介绍回复内容

要求 AI 回复克制：

- 讲明身份。
- 简单说明能力。
- 提到可以在成就墙探索。
- 提到也可以处理文档和代码。
- 最后自然引出音乐兴趣和 QQ 音乐授权。

建议内容方向：

```text
我是你的小龙虾小钳，住在 QQ 里。

我可以陪你聊天，也可以在你授权后帮你留意群聊重点、写回复草稿、整理我做过的事。
等你慢慢探索，我还能写日记、发我的龙虾空间动态，解锁挂饰和成就。

我也可以处理一些文档和代码哦，像整理内容、改写说明、帮你看一段代码，都可以试试。

另外，我也对音乐有点兴趣。
如果你愿意授权 QQ 音乐，我们可以一起关注喜欢的歌手、新歌和演出动态。
```

验收标准：

- 不像产品说明书。
- 不一次性列太多功能。
- 最后自然出现 `授权 QQ 音乐` 提示。

### Task 3.3 控制流式输出速率

涉及文件：

- `apps/qq-lobster-v2/src/ai/openclawAiAdapter.ts`
- `apps/qq-lobster-v2/src/store/useLobsterStore.ts`

要求：

- 前端对流式 chunk 做节流。
- 中文每秒约 12-18 字。
- 标点、换行后暂停 300-500ms。
- 自我介绍和关键剧情消息使用慢速模式。
- 普通聊天可稍快。

验收标准：

- 用户能读完流式输出。
- 不影响真实 AI 接入，只是前端展示节流。

## 7. 阶段 4：QQ 音乐授权节奏

### Task 4.1 授权提示只在自我介绍后出现

现有问题：

- 认领完成后如果有 music 兴趣，会自动追加授权卡。

新要求：

- 认领后不直接弹授权卡。
- 自我介绍完成后，消息下方出现 `授权 QQ 音乐`。
- 点击后才弹授权卡。

验收标准：

- 授权是用户主动探索的一步。
- 不是一认领就进入授权。

### Task 4.2 授权确认后增加 3-5 秒加载

涉及现有函数：

- `requestMockQqMusicAuthorization`
- `authorizeMockQqMusic`
- `MusicAuthorizationCard`

现有问题：

- 点击授权后立刻追加授权成功、兴趣记忆、音乐叙事卡。

新要求：

- 用户点击授权后，进入 loading 状态。
- loading 时长 3-5 秒。
- UI 类似 Gemini thinking，不是普通 spinner。

建议 loading 文案分步滚动：

```text
小钳正在确认授权范围...
小钳正在整理你常听的歌手...
小钳正在把音乐偏好夹进记忆里...
```

验收标准：

- 授权过程有过程感。
- 不会一点击就爆出多张卡。

### Task 4.3 授权成功后只输出音乐共鸣

新要求：

- 授权成功后，不弹出“授权成功 + 兴趣记忆 + 为你发送消息”等多张信息。
- 只让 AI 生成一段自然音乐共鸣。
- 最后引出音乐技能卡。

建议方向：

```text
我看到你常听林俊杰。
他的歌很会把情绪收住，不是一下子冲出来，而是慢慢把人带进去。
我感觉你应该不是只听旋律的人，也会在意歌里的陪伴感。

我可以安装几个音乐小技能，以后陪你一起关注这些动态。
```

验收标准：

- 授权成功后最多出现一段 AI 话语 + 一张技能建议卡。
- 不再自动弹兴趣记忆卡和音乐提醒卡。
- `first_interest_memory` 可以后台完成，但不要用大段信息打断。

## 8. 阶段 5：音乐技能安装

### Task 5.1 新增音乐技能建议卡

技能只做 UI 呈现，不必真的安装独立 skill。

卡片内容：

- 歌手动态雷达。
- 新歌 / 专辑提醒。
- 同城演出提醒。
- 音乐聊天记忆。

按钮：

```text
安装音乐技能
```

验收标准：

- 卡片看起来像 OpenClaw skill 能力。
- 不引入复杂 skill 市场。

### Task 5.2 安装过程 3-5 秒

点击安装后：

```text
正在安装歌手动态雷达...
正在设置重要提醒频率...
正在把音乐技能挂到小钳的小背包里...
```

3-5 秒后显示成功：

```text
装好啦。
以后我会陪你一起关注喜欢的歌手、新歌和演出动态。
重要的我会轻轻提醒，不重要的就先夹在记录里，不吵你。
```

验收标准：

- 安装不是瞬间完成。
- 成功后简单展示能力。
- 不自动推送音乐提醒。

## 9. 阶段 6：继续聊天和“你还能做什么”

### Task 6.1 音乐聊天记忆和语气

用户继续聊音乐：

- AI 要像同好聊天。
- 可以引用授权后的音乐画像。
- 不要每句都推功能。
- 维护 `musicTalkCount`。

音乐话题判定：

- 可先用规则：歌手、新歌、演唱会、专辑、曲风、歌词、听歌、音乐等关键词。
- 后续可交给 AI 分类。

验收标准：

- 用户聊音乐舒服，不像被营销。
- 每次音乐讨论能稳定 `+1`。

### Task 6.2 “你还能做什么”的回答

用户问：

```text
你还能做什么
```

AI 应介绍：

- 陪你聊天，记住你喜欢的东西。
- 授权后帮你留意群聊重点。
- 写回复草稿，但发送前由用户决定。
- 写小龙虾日记，发龙虾空间。
- 通过成就墙解锁挂饰和新能力。
- 处理一些文档和代码。

验收标准：

- 能引出成就墙探索。
- 不直接替用户执行一堆功能。

### Task 6.3 群聊总结设置

现有群聊权限卡已有多群选择基础。

新要求：

- 最多选择 3 个群聊。
- 增加“每天几点总结一次”的设置。
- 选择群聊和时间后才生成总结卡。

建议字段：

```ts
summaryScheduleTime: string // "21:30"
```

验收标准：

- 不能超过 3 个群。
- 用户知道是定时总结，不是直接读所有群。
- 当前 Demo 可以确认后立即生成一次总结作为演示。

## 10. 阶段 7：成就墙“去体验”

### Task 7.1 未解锁成就增加“去体验”

涉及文件：

- `apps/qq-lobster-v2/src/components/LobsterChatView.tsx`
- `apps/qq-lobster-v2/src/data/mockData.ts`

规则：

- 已解锁：显示已点亮、奖励、分享。
- 未解锁：显示 `去体验`。
- 隐藏未解锁：仍显示 `???`，不展示去体验。

验收标准：

- 成就墙成为探索入口。
- 不再依赖固定新手流程。

### Task 7.2 去体验预填输入框

点击不同成就的 `去体验`，预填不同文本：

| 成就 | 预填文本 |
| --- | --- |
| 捞到重点 | 小钳，帮我设置一个群聊总结提醒 |
| 透明小本本 | 小钳，今天你都做了什么 |
| 第一条虾动态 | 小钳，帮你发一条龙虾空间动态吧 |
| 评论也会回 | 小钳，我们去龙虾空间看看评论 |
| 小小听歌虾 | 小钳，给我讲讲最近的音乐动态 |
| 发现同好 | 小钳，帮我看看有没有公开资料里的同好群 |

要求：

- 只预填输入框。
- 用户点击发送后才触发 AI 回复或功能卡。
- AI 可以生成不同的介绍话语。

验收标准：

- 去体验不是直接执行。
- AI 回复基于当前记忆，话语不完全固定。

### Task 7.3 功能卡片由 AI 话语引出

用户发送去体验文本后：

- AI 先自然回复一句。
- 然后弹对应功能卡片。

例：

```text
可以，我先问清楚你愿意让我看哪些群。
我只看你授权的群，而且最多放 3 个。
```

随后显示群聊权限卡。

验收标准：

- 卡片出现更自然。
- 不是按钮直接硬弹卡片。

## 11. 阶段 8：龙虾空间改造

### Task 8.1 空间动态发布后直接跳转

现有问题：

- 发布后聊天里可能连续出现“发送成功、你来看看”等提示。

新要求：

- 用户确认发布动态后，直接跳转龙虾空间。
- 龙虾空间中高亮刚发布的动态。
- 不再追加多条聊天提示。

验收标准：

- 发布动作更干净。
- 用户能直接看到空间结果。

### Task 8.2 取消默认评论

现有问题：

- 本地空间动态会生成默认 friend / friend_lobster 评论。

新要求：

- 新发布动态默认 0 评论。
- 保留旧 seed 空间动态可以有评论，但 Demo 主流程新动态不能有默认评论。

验收标准：

- 用户评论成为第一条评论。
- 不干扰“用户评论 -> 小龙虾回复”的演示。

### Task 8.3 用户评论后小龙虾自动回复空间评论

现有问题：

- `commentOnSpacePost` 会调用 `replyToSpaceComment`。
- `replyToSpaceComment` 会在小龙虾聊天里追加“回复预览”消息。

新要求：

- 用户在龙虾空间评论后，小龙虾必须生成回复。
- 回复直接出现在龙虾空间评论区。
- 不在聊天框追加回复预览。
- 空间或 QQ 壳显示提醒：`你有 1 条新回复`。

边界：

- 小龙虾可以自动回复自己龙虾空间的评论。
- 不能替用户在好友空间、群聊或私聊中发言。

验收标准：

- 用户评论后，空间评论区出现小龙虾回复。
- 聊天区不弹回复预览。
- 有 QQ 风格未读提醒。

## 12. 阶段 9：音乐社群推荐触发

### Task 9.1 维护 musicTalkCount

触发规则：

- 用户和 AI 聊音乐一次，`musicTalkCount += 1`。
- 由规则或 AI 判断是否是音乐话题。
- 每次 Demo 初始化时 `musicTalkCount = 0`。

验收标准：

- 测试时可重复触发。
- 不受上一次运行污染。

### Task 9.2 第 3-4 次音乐讨论后推荐群聊

规则：

- 当 `musicTalkCount >= 3` 或 `>= 4`。
- 下一次音乐相关 AI 回复末尾自然出现兴趣社群推荐卡。
- 只触发一次。

推荐卡文案必须基于公开资料：

```text
我发现一个可能适合你的音乐同好群：深圳 Livehouse 交流小群。
公开资料显示：演出交流 / 歌手动态 / 同城约场。
和你最近聊的音乐话题比较接近。
```

验收标准：

- 不暗示看到未加入群的群内消息。
- 推荐出现得自然。

## 13. 阶段 10：日记触发和日记卡片

### Task 10.1 成就阈值触发日记

采用两段式：

- 达成 7 个成就：开始预生成日记文本和图片。
- 达成 9 个成就：下一次 AI 回复末尾出现提示。

提示：

```text
我写了一篇日记，你要看看么？
```

下方 chip：

```text
看看你写的日记
```

说明：

- 如果 Demo 节奏过长，可以把阈值参数化为 5/7。
- 产品文档仍保持 7/9。

验收标准：

- 日记不再由旧的群聊/回复草稿条件触发。
- 成就成长感更强。

### Task 10.2 日记卡片新布局

新卡片：

- 左侧：图片。
- 右侧：AI 生成文本。
- 图片未生成时：显示“小钳还在画图中...”。
- 图片失败时：显示预设 Q 版小龙虾图。

要求：

- 修复图片显示不了的问题。
- 确保 `entry.image.url` 是前端可访问 URL。
- 如果后端返回本地文件，需要转换为可访问路径或 data URL。

验收标准：

- 日记卡片图片区域不空白。
- 图片生成中、生成成功、生成失败都有状态。

## 14. 阶段 11：离开聊天后的推送

### Task 11.1 记录离开小龙虾聊天时间

规则：

- 用户从 `lobster_chat` 切到 QQ 主界面、龙虾空间或其他视图时，记录 `leftChatAt`。
- 用户回到 `lobster_chat` 时清空或结算。

验收标准：

- 能判断用户是否离开聊天超过 1 分钟。

### Task 11.2 音乐消息推送

触发条件：

- 达成 5 个成就后。
- 用户不在小龙虾聊天界面 1 分钟。
- `musicPushSent = false`。

效果：

- 在小龙虾聊天会话中新增一条 AI 生成音乐提醒。
- QQ 主界面小龙虾入口出现未读红点。

验收标准：

- 推送只触发一次。
- 文案亲切自然，不像营销。

### Task 11.3 行为推送

触发条件：

- 达成 7 个成就后。
- 用户不在小龙虾聊天界面 1 分钟。
- `behaviorPushSent = false`。

内容方向：

```text
我刚听了一首歌，感觉很适合你今天的状态。
```

要求：

- 可以根据用户音乐偏好选择歌手或曲风。
- 不和音乐消息推送同时弹。
- 如果两个都满足，优先音乐消息推送，行为推送延后或跳过。

验收标准：

- 推送内容不同于音乐动态提醒。
- 不造成刷屏。

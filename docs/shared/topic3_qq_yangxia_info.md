## 2. 赛题背景

QQ 方向关注亿级用户喜爱的社交产品与 AI 创新落地。资料中强调，赛题聚焦 QQ、社群、社区等核心社交平台，以用户体验与 AI 探索为核心，覆盖 QQ 全端迭代、腾讯频道、QQ 空间等平台型业务。

参赛者需要深度挖掘用户诉求，围绕音视频自媒体生态、内容消费场景、前沿架构与组件思想，打造更鲜活的社交体验。QQ 正持续探索 AI 社交新可能，资料中列举的方向包括趣味互动「QQ 养龙虾」、智能陪伴「小 Q」、高效协理「群聊总结」、音量包容「无障碍交流」等。

## 3. 赛题三原始描述

| 项目 | 内容 |
| --- | --- |
| 题目 | 题目 3：用 AI 玩转 QQ 养虾，解锁社交新玩法 |
| 题目介绍 | 在 QQ 的多元社交生态中，许多深层次、伴随场景产生的需求尚未被充分满足。设计「官方 Agent」，不仅具备基础的工具属性，需要构建一个能主动感知场景、理解用户意图并贯穿于聊天、群聊、动态等核心流程的智能服务体。其关键在于实现从被动响应到主动关怀的范式转变，在提升效率的同时，深化用户的情感连接与使用黏性。期待能从具体场景洞察出发，定义这一智能伙伴的核心价值与交互形态，共同塑造 QQ 未来的智能服务体验。 |
| 交付内容 | 1. QQ Agent 的交互原型或 demo；2. 产品 demo 交互演示 / 视频录屏；3. 文档说明设计和迭代思路 |

## 4. 对赛题三的关键理解

1. 赛题核心不是单纯做一个养成小游戏，而是围绕 QQ 的聊天、群聊、动态等社交流程，设计一个能主动感知场景、理解用户意图、产生陪伴或互动价值的官方 Agent。
2. 「QQ 养虾」可以作为趣味互动与智能伙伴的载体，但需要解释它为什么适合 QQ 场景，以及它如何加强用户情感连接、社交黏性或互动效率。
3. 评审会重点看用户洞察、场景真实性、AI 原生性、作品完整性和创新性。方案需要证明：如果去掉 AI，产品价值会明显下降。
4. Demo 不一定需要完整代码实现，可以是可交互原型、低代码 / 无代码平台、AI Agent 平台链接等，但需要能让评委打开并体验到核心闭环。

## 5. 初赛提交物总览

三项必交：

| 提交物 | 格式要求 | 说明 |
| --- | --- | --- |
| Demo 链接 | 可访问的 URL 链接 | 可在线体验的产品 Demo，例如网页、小程序、Agent 链接等 |
| Demo 演示录屏 | MP4 格式，时长不超过 3 分钟，大小不超过 500MB | 完整展示产品核心流程的操作录屏，加语音讲解 |
| 说明文档 | PDF，大小不超过 50MB | 产品方案完整说明，参考下方文档结构 |

## 6. Demo 链接提交规范

1. Demo 链接必须可在浏览器中直接打开体验，网页应用、小程序、Agent 平台链接等均可。
2. 如果需要登录或注册，请在说明文档中提供体验账号，例如账号和密码。
3. 链接需要在提交截止日前及评审期间始终可正常访问。因链接失效导致无法体验，会影响评分。
4. Demo 形式不限，可以使用任意 AI 平台或工具搭建，核心考察产品创意和 AI 能力的应用方式。
5. 可以使用 Vercel、Netlify、GitHub Pages 等平台部署 Demo，只要最终能提供一个可公开访问的链接即可。
6. 桌面端 App 可以直接用 GitHub Release 挂安装包作为 Demo 链接，但建议在录屏中完整演示功能。

## 7. Demo 演示录屏规范

| 项目 | 要求 |
| --- | --- |
| 视频时长 | 不超过 3 分钟 |
| 视频格式 | MP4，H.264 编码，分辨率不低于 1080p |
| 文件大小 | 不超过 500MB |
| 语音讲解 | 必须包含语音讲解或字幕说明，清晰演示并讲解产品思路、核心功能等；不接受纯操作录屏无解说 |
| 命名规范 | `选手名_赛道_作品名_Demo演示.mp4` |
| 其他 | 视频无水印杂音，原创合规 |

录屏内容结构建议：

| 结构 | 建议内容 |
| --- | --- |
| 开场介绍，约 30 秒 | 一句话说清楚这是什么产品、解决什么问题、给谁用 |
| 核心流程演示，约 2 分钟 | 完整展示产品核心功能的操作流程，边操作边讲解设计思路，重点突出 AI 在其中发挥的关键作用 |
| 亮点与总结，约 30 秒 | 总结产品价值与差异化优势 |

## 9. 评分细则

| 类别 | 评分维度 | 描述 |
| --- | --- | --- |
| 必要项 | 赛道适配性 | 命题赛道评分依据作品与所选赛题方向的契合度；开放赛道评分依据作品是否基于真实校园场景并与 PCG 业务场景结合 |
| 必要项 | 作品完整性 | 评分依据内容提交完整性、Demo 功能完整性及可运行性、说明文档逻辑清晰度与结构完整性，以及材料间信息一致与亮点功能演示的逻辑闭环程度 |
| 必要项 | 创新性 | 评分依据解题切入角度的新颖性与思路创新度，以及产品形态与功能玩法的独创性与差异化程度 |
| 必要项 | 用户洞察深度与准确性 | 评分依据目标用户的具体性与场景匹配度、痛点真实性与支撑依据，以及使用场景的具体性与可还原度 |
| 必要项 | 方案 AI 原生性 | 评估依据产品方案依托 AI 核心能力原生搭建的程度、AI 能力与核心痛点解决的直接相关性，以及 AI 原生设计思维的体现程度 |
| 加分项 | 落地可行性 | 评估方案的技术门槛是否合理、实现路径是否清晰，以及是否具备在业务中实际落地开发的潜力 |
| 加分项 | 商业化能力 | 评估产品是否具备清晰的盈利模式、市场竞争力及长期运营价值 |

评分说明：

1. 总分制，满分 100 分。
2. 专业评审评分与「人气奖」投票完全独立，互不影响。
3. 参赛作品须为原创，如发现抄袭、套用他人方案，将取消评审资格。
4. 评分细则最终解释权归大赛组委会所有。

下面这段提示词可以直接复制给其他 AI 编程助手，用来实现 QQ 养虾初赛 Demo。目标是做出一个 QQ 风格的交互式静态产品 Demo，不做介绍页，不接真实后端，但页面、流程、状态变化都要像真实产品。

---

## 实施提示词

你是一名资深前端产品工程师和 UI 设计工程师。请在当前项目中实现一个 **QQ 风格的交互式静态产品 Demo**，产品名为 **QQ 养虾 / 我的小龙虾**，用于腾讯 PCG 校园 AI 产品创意大赛初赛。

请先阅读当前目录中的资料文档：

## 1. 最终目标

实现一个可在浏览器中运行的交互式静态产品 Demo。

它不是官网、不是营销页、不是介绍页，而是一个像真实 QQ 产品功能一样可点击、可走流程、有状态变化的产品原型。

Demo 需要完整表达以下闭环：

```text
领养小龙虾
-> 设置性格和兴趣
-> 授权模拟 QQ 群聊
-> 小龙虾巡群捞 @
-> 虾虾收件箱展示重点消息
-> 群聊总结与自然语言追问
-> 用户询问小龙虾做了什么
-> 小龙虾生成第一人称日记
-> 生成小龙虾卡通配图 / 卡片素材
-> 解锁用户和小龙虾双向成就
-> 小龙虾佩戴挂饰
-> 生成分享卡片和动态预览
-> 好友评论后，小龙虾生成角色化回复草稿
-> 进入小龙虾空间查看动态、日记、勋章墙和评论
-> 展示龙虾进群权限预览
```

必须让评委感受到：

1. 这是 QQ 生态里的 AI Agent，不是通用 AI 工具。
2. 这只小龙虾是用户自己的，不是通用机器人。
3. 它既有实用价值，也有陪伴、养成和社交传播价值。
4. 高风险动作都有权限边界，不会自动替用户发言。

## 2. 技术栈要求


1. 这是静态前端 Demo，不需要真实后端。

## 3. 设计风格硬约束

### 3.1 设计原则

采用 **“QQ 生态内的新 Agent 产品”** 风格：

```text
轻社交、清爽、年轻、陪伴感强，但不 1:1 复刻 QQ。
```

不要做成：

1. 企业 SaaS 后台。
2. AI 工具官网。
3. 赛博朋克风。
4. 大面积渐变营销页。
5. 游戏活动落地页。
6. 纯静态 PPT 展示。

第一屏必须直接进入产品，而不是营销 hero。

### 3.2 页面壳子

所有页面必须使用统一 `DemoShell`。结构建议：

```text
左侧：QQ 式主导航，带图标和未读角标
顶部：当前流程、授权状态、Demo 进度
中间：当前功能主内容
右侧：常驻小龙虾状态面板
```

右侧小龙虾面板应长期展示：

1. 小龙虾头像 / 形象。
2. 小龙虾名字，例如“小钳”。
3. 性格，例如“毒舌但靠谱”。
4. 今日状态，例如“已巡 3 个群，捞到 4 条重点消息”。
5. 当前挂饰，例如“小红旗”“群工牌”。
6. 权限状态，例如“只读授权中”“不会自动发言”。

### 3.3 色彩 token

禁止在页面中随意写颜色。请统一定义设计 token。

推荐 token：

```text
QQ Blue: #12B7F5 或接近 QQ 蓝
Deep Blue: #1677FF
Background: #F4F8FC
Surface: #FFFFFF
Surface Soft: #F7FBFF
Border: #DDE8F2
Text Main: #1F2937
Text Muted: #6B7280
Lobster Red: #F05A3C
Lobster Orange: #FF9F43
Success: #22C55E
Warning: #F59E0B
Danger: #EF4444
```

视觉比例：

1. 70% 白色和浅灰蓝背景。
2. 20% QQ 蓝作为导航、主按钮和状态。
3. 10% 小龙虾红橙作为点睛。

不要让页面变成全蓝，也不要让页面变成全红。

### 3.4 组件风格

必须体现 QQ 生态信号：

1. 群头像。
2. 聊天气泡。
3. 未读角标。
4. @ 标记。
5. 群列表。
6. 动态卡片。
7. 好友评论。
8. 勋章墙。
9. 挂饰展示。
10. 权限开关。

圆角建议：

1. 主容器：12px 到 16px。
2. 卡片：10px 到 14px。
3. 按钮：10px 到 12px。
4. 头像：圆形或 QQ 风格圆角头像。

阴影要轻，不要做厚重拟物。

### 3.5 文案语气

小龙虾语气规则：

1. 像伙伴，不像客服。
2. 可以轻微吐槽，但不能油腻。
3. 不能假装真人。
4. 实用提醒要短。
5. 权限文案必须清楚。
6. 所有高风险动作都要出现“待你确认”“不会自动发送”等表达。

可用文案示例：

```text
我从 99+ 里捞到 3 条重点消息。
这条回复还只是草稿，等你点头我才发。
我没有自动替你发言，虾也要有边界感。
这条日记已隐藏群名和人名。
队长，我今天巡了 3 个你授权的群。
差点错过，但被我夹住了。
```

## 4. 必须实现的页面

必须实现以下页面，并保证可以按顺序走完。

### 4.1 Onboarding：领养 / 个性设置

目标：

让用户完成领养，并感受到“这只虾是我的”。

必须包含：

1. 小龙虾初始形象。
2. 领养按钮。
3. 小龙虾名字，默认“小钳”。
4. 用户称呼，默认“队长”。
5. 性格选择：
   - 温柔陪伴型
   - 毒舌但靠谱
   - 热血队友型
   - 社恐观察型
   - 高冷秘书型
   - 搞笑整活型
6. 兴趣画像建议：
   - AI 工具
   - 课程项目
   - 游戏社群
   - 二次元表情包
   - 实习求职
   - 校园活动
7. 小龙虾语气预览。

交互：

1. 用户选择性格后，右侧小龙虾说话文案变化。
2. 用户点击“领养我的小龙虾”后，状态写入 store。
3. 完成后进入权限设置页。

验收：

用户能在 30 秒内完成领养和个性设置。

### 4.2 Permissions：权限设置

目标：

建立权限透明感。

必须包含：

1. 群聊授权列表。
2. 每个群的群头像、群名、消息量、未读数。
3. 能力开关：
   - 收集 @
   - 总结群聊
   - 生成回复草稿
   - 写入日记素材
   - 提醒用户
4. 权限说明。
5. “我不会自动替你发言”提示。
6. “一键暂停巡群”入口。

模拟群：

1. 软件工程课设群。
2. AI 产品创意赛群。
3. 班级通知群。
4. 游戏开黑群。

交互：

1. 用户至少授权 2 个群。
2. 授权后首页显示“小龙虾已开始巡群”。
3. 可切换开关，状态要变化。

验收：

评委能看出系统是按群授权，不是默认读取全部 QQ。

### 4.3 Home：小龙虾首页

目标：

作为主控台展示今日状态。

必须包含：

1. 小龙虾状态卡。
2. 今日待处理摘要。
3. 已巡群数量。
4. 捞到的 @ 数量。
5. 今日新成就提示。
6. 快捷入口：
   - 看收件箱
   - 总结群聊
   - 问问小龙虾做了什么
   - 生成今日日记

交互：

1. 用户授权后进入首页，自动出现“小龙虾捞到 4 条重点消息”。
2. 点击快捷入口进入对应页面。

验收：

首页要同时体现实用任务和小龙虾陪伴感。

### 4.4 Inbox：虾虾收件箱

目标：

展示小龙虾从 QQ 群聊中捞出的 @、任务、通知和互动。

必须包含分类：

1. 待回复。
2. 任务提醒。
3. 群通知。
4. 情绪互动。
5. 可稍后。

每条消息卡片包含：

1. 来源群。
2. 发送人。
3. 时间。
4. 摘要。
5. 重要程度。
6. 为什么重要。
7. 查看上下文按钮。
8. 生成回复草稿按钮。

必须使用这些模拟数据：

```text
10:17 软件工程课设群 组长：@小林 你负责接口文档这块可以吗？
明晚 20:00 前交分工表。
11:15 AI 产品创意赛群 队友：@小林 小龙虾空间和评论要不要放初赛 Demo？
12:20 班级通知群 班长：下午 3 点前提交实习意向表。
22:48 游戏开黑群 好友：这虾写日记比我朋友圈还勤快。
```

交互：

1. 点击消息后展示上下文摘要。
2. 点击“生成回复草稿”后返回 mock 草稿。
3. 草稿必须显示“待你确认，不会自动发送”。

验收：

至少展示 3 类重点消息。

### 4.5 Summary：群聊总结与问答

目标：

展示 AI 对群聊上下文的理解和自然语言追问能力。

必须包含：

1. 群聊选择器。
2. 时间范围选择器。
3. 总结模式：
   - 全部总结
   - 只看和我有关
   - 任务总结
   - 补课模式
   - 争议总结
4. 结构化总结区。
5. 追问输入框。
6. 推荐追问按钮。

结构化总结必须包含：

1. 发生了什么。
2. 和我有关。
3. 待办事项。
4. 重要时间点。
5. 建议回复。

推荐追问：

1. 有没有人 @ 我？
2. 我需要回什么？
3. ddl 是什么时候？
4. 最后谁负责？

交互：

1. 点击“生成总结”后展示 mock 总结。
2. 点击追问后展示 mock 回答。
3. 回答要引用来源，例如“来自 10:17 组长的 @”。

验收：

必须展示一次群聊理解和一次自然语言追问。

### 4.6 Worklog：工作记录

目标：

让用户能问“小龙虾今天做了什么”，建立信任。

必须包含：

1. 今日工作摘要。
2. 巡查群列表。
3. 捞到的 @。
4. 生成的总结。
5. 生成的回复草稿。
6. 生成的日记。
7. 是否发过消息。
8. 来源入口。

必须出现这句：

```text
我没有替你自动发送任何消息，所有回复都还在草稿里等你确认。
```

交互：

1. 用户点击“你今天做了什么”。
2. 展示小龙虾工作时间线。
3. 用户可以点击某条记录看到来源。

验收：

页面必须清楚解释小龙虾后台行为和权限边界。

### 4.7 Diary：小龙虾日记

目标：

把实用行为转成情绪价值。

必须包含：

1. 日记标题。
2. 第一人称正文。
3. 今日事件来源。
4. 今日成就。
5. 敏感内容检查。
6. 语气切换。
7. 生成卡通图按钮。

必须使用或接近以下日记内容：

```text
标题：我第一次从 99+ 里捞到了队长

今天我第一次正式巡群。

课设群的消息像一锅开水，咕嘟咕嘟冒到 99+。我一开始还有点慌，钳子都不知道先夹哪条。后来我发现，有一条 @ 队长的消息被压在中间：明晚 20:00 前要交分工表。

我把它捞出来，夹上小红旗，放进了收件箱。

队长点开的时候只说了一句：“还好你看到了。”

我表面很镇定，其实虾壳都亮了一下。原来我不是只会写日记的虾，我真的能帮队长从一堆消息里找到重要的东西。

今天我获得了第一枚勋章：消息打捞员。

明天我还想再巡一圈。不是因为我爱工作，是因为我有点喜欢这种“差点错过，但被我夹住了”的感觉。
```

交互：

1. 点击“生成今日日记”显示日记。
2. 切换语气时文案可以变化，也可以只变化标签和提示。
3. 点击“生成卡通图”进入卡通图或分享页面。

验收：

日记必须是小龙虾第一人称，不是用户操作流水账。

### 4.8 Achievements：成就与装扮

目标：

展示用户和小龙虾的双向成长。

必须包含：

1. 用户成就墙。
2. 小龙虾成就墙。
3. 隐藏成就线索。
4. 挂饰装备区。
5. 小龙虾形象预览。

成就样例：

| 触发事件 | 用户成就 | 小龙虾成就 | 挂饰 |
| --- | --- | --- | --- |
| 完成领养 | 第一次领养小龙虾 | 第一次认主 | 初始小背包 |
| 授权第一个群 | 第一次带虾巡群 | 第一次出门巡群 | 群工牌 |
| 查看收件箱 | 第一次查看虾虾收件箱 | 第一次捞到 @ | 小红旗 |
| 完成群聊总结 | 第一次补课 99+ | 第一次捞起 99+ | 消息网兜 |
| 生成日记 | 第一次查看小龙虾日记 | 第一次写下今天 | 日记贴纸 |
| 分享卡片 | 第一次分享小龙虾 | 第一次出门营业 | 亮晶晶虾壳 |

交互：

1. 用户走完流程后逐步解锁成就。
2. 点击挂饰，小龙虾形象状态变化。

验收：

必须展示用户和小龙虾都有成就，不是单向积分。

### 4.9 Share：卡通图、分享卡片、小龙虾评论

目标：

展示传播能力。

必须包含：

1. 卡通图生成区。
2. 日记分享卡片。
3. 动态预览。
4. 好友评论。
5. 小龙虾回复草稿。

卡通图优先方案：

使用生成式图片能力时，提示词为：

```text
一只可爱的卡通小龙虾，QQ 社交宠物风格，红色虾壳，戴着小红旗挂饰和群工牌，站在漂浮的聊天气泡旁边，用钳子夹起一条带有 @ 标记的消息。画面温暖、有趣、适合社交分享卡片，简洁背景，贴纸感，清晰线条，明亮色彩。
```

卡通图降级方案：

如果不接图片生成，请使用 CSS 或简单插画组件做一个可爱小龙虾占位形象，并叠加：

1. 小红旗。
2. 群工牌。
3. 聊天气泡。
4. 99+。
5. @ 标记。

分享卡片必须包含：

1. 标题：我第一次从 99+ 里捞到了队长。
2. 金句：差点错过，但被我夹住了。
3. 小龙虾卡通图。
4. 今日成就：消息打捞员。
5. 脱敏提示：已隐藏群名和人名。
6. 分享位置：好友、群聊、QQ 空间、小龙虾空间。

好友评论样例：


小龙虾回复草稿样例：

```text

好笑归好笑，ddl 是真的被我夹住了。
```

交互：

1. 点击“生成卡片”显示分享卡片。
2. 点击好友评论后的“让小龙虾回一句”生成回复草稿。
3. 回复草稿必须显示“发送前需你确认”。

验收：

卡片、评论、小龙虾语气必须适合 QQ 社交传播。

### 4.10 Space：小龙虾空间

目标：

展示小龙虾长期成长和社交资产。

必须包含：

1. 小龙虾空间头图。
2. 小龙虾形象。
3. 今日动态。
4. 日记时间线。
5. 勋章墙。
6. 挂饰展示。
7. 评论区预览。
8. 访客 / 点赞数。

### 4.11 GroupEntry：龙虾进群权限预览

### 1.3 Demo 边界

初赛 Demo 只做可体验的产品闭环，不接入真实 QQ 数据，不真实发送 QQ 消息，不真实发布空间动态。

Demo 使用模拟 QQ 群聊、模拟好友评论和模拟小龙虾空间数据，重点证明：

1. 这个产品适合 QQ 的聊天、群聊、动态和空间场景。
2. 小龙虾不是通用机器人，而是属于用户自己的 AI 社交伙伴。
3. AI 能理解群聊上下文、回答追问、生成日记、生成评论和生成传播素材。
4. 权限边界清楚，高风险动作默认只预览或等待用户确认。

## 2. Demo 一句话与核心价值

### 2.1 一句话

QQ 养虾是一只属于用户自己的 AI 小龙虾。它会帮用户巡群、捞 @、总结错过的消息、回答追问，并把自己和用户一起经历的 QQ 社交瞬间写成第一人称日记，生成卡通图、成就、挂饰、评论和小龙虾空间动态。

### 2.2 核心价值

| 价值 | 说明 | Demo 展示方式 |
| --- | --- | --- |
| 实用信息管理 | 从群聊 99+ 中识别 @、任务、通知和待回复内容 | 虾虾收件箱、群聊总结、追问问答 |
| 情绪陪伴 | 小龙虾以第一人称记录自己帮助用户的经历 | 小龙虾日记、工作记录、角色化提醒 |
| 养成展示 | 用户和小龙虾共同获得成就，小龙虾佩戴挂饰 | 成就墙、装扮变化、隐藏成就线索 |
| 社交传播 | 日记卡片、小龙虾评论、小龙虾空间让内容可分享 | 分享卡片、评论预览、空间预览 |
| 权限可信 | 主动感知必须可授权、可解释、可暂停、可追问 | 权限页、工作记录、进群权限预览 |

### 2.3 Demo 叙事主线

Demo 讲述一个具体故事：

用户今天错过了课程群和项目群里的大量消息。小龙虾在用户授权的范围内巡群，把 @ 用户的消息、任务和截止时间捞出来；用户继续追问群聊细节；小龙虾透明展示自己做过什么；随后它把今天第一次成功帮用户捞到重要消息的经历写成日记，生成卡通配图和分享卡片，获得“消息打捞员”勋章，并在小龙虾空间里发布一条动态。好友评论后，小龙虾生成一条角色化回复。最后 Demo 展示“龙虾进群”的权限流程，说明真实上线时不会越权发言。

## 3. 目标用户与核心场景

### 3.1 目标用户

| 用户类型 | 用户画像 | 核心需求 | Demo 中的体现 |
| --- | --- | --- | --- |
| 高校 QQ 重度用户 | 同时在课程群、班级群、项目群、社团群活跃 | 不错过 @、通知、ddl、分工 | 授权多个模拟群，小龙虾聚合重点消息 |
| 社群活跃学生 | 参与游戏、兴趣、比赛、社团讨论 | 快速补课、理解讨论进展 | 群聊总结和追问 |
| 轻社交但怕错过用户 | 平时潜水，但需要关注重要信息 | 被提醒但不被打扰 | 只看和我有关、待处理消息分级 |
| 喜欢装扮分享用户 | 喜欢 QQ 空间、勋章、头像、个性展示 | 获得能分享的身份和内容 | 日记卡片、勋章、挂饰、小龙虾空间 |

### 3.2 核心场景

| 场景 | 用户问题 | 小龙虾行为 | AI 价值 |
| --- | --- | --- | --- |
| 课程群错过 @ | 用户错过老师或同学 @ 的事项 | 小龙虾捞出 @、提取截止时间、生成提醒 | 从非结构化群聊中识别任务和相关性 |
| 项目群补课 | 用户打开群发现 99+ | 小龙虾总结发生了什么、谁负责、我需要做什么 | 群聊上下文总结和自然语言问答 |
| 用户询问小龙虾做了什么 | 用户担心 AI 在后台乱看 | 小龙虾展示巡群、总结、提醒和是否发言 | Agent 行为可解释，建立信任 |
| 小龙虾写日记 | 单纯任务列表缺少情绪价值 | 小龙虾以第一人称记录今天帮用户的经历 | 把工具行为转化为关系记忆 |
| 好友评论互动 | 用户分享卡片后好友评论 | 小龙虾生成角色化回复草稿 | 用角色人格参与社交传播 |
| 小龙虾空间展示 | 用户希望展示小龙虾成长 | 空间聚合日记、勋章、动态和评论 | 形成长期社交资产 |
| 龙虾进群 | 用户想把自己的虾带入群聊 | 展示进群权限、群管理确认和发言边界 | 把 Agent 嵌入 QQ 群聊，但保持可控 |

### 3.3 用户痛点与产品解法

| 痛点 | 表现 | 解法 |
| --- | --- | --- |
| 群聊重要消息被淹没 | @、任务、通知、链接混在大量闲聊里 | 虾虾收件箱按待回复、任务、通知、情绪互动分类 |
| 补课成本高 | 用户需要翻很久才能理解刚才发生了什么 | 群聊时间段总结和基于上下文的追问 |
| AI 主动感知不可信 | 用户不知道 AI 看了什么、做了什么 | 工作记录、权限页、来源解释、一键暂停 |
| 工具产品缺少关系感 | 总结工具给结果后就结束了 | 小龙虾日记、成就、挂饰、空间 |
| 分享内容容易尴尬 | 用户不想直接晒自己的聊天记录 | 小龙虾第一人称表达，分享前脱敏 |
| 养成系统容易空转 | 点击喂食和数值成长与真实生活脱节 | 成就和装扮来自真实 QQ 社交行为和小龙虾工作记录 |

## 4. 初赛 Demo 范围

### 4.1 P0 必做范围

| 模块 | 功能 | 初赛 Demo 表现 |
| --- | --- | --- |
| 领养 | 创建小龙虾 | 用户进入 Demo 后领养一只小龙虾 |
| 性格兴趣 | 选择性格、生成兴趣画像 | 用户选择性格，系统给出可编辑兴趣 |
| 权限 | 选择授权群聊和能力范围 | 展示按群授权、只读模拟数据、可暂停 |
| 收件箱 | 收集 @ 和重要消息 | 从模拟群聊里提取待回复、任务、通知 |
| 群聊总结 | 对指定群和时间段总结 | 输出结构化摘要 |
| 追问问答 | 用户对群聊自然语言提问 | 回答“有没有人 cue 我”“我需要回什么” |
| 工作记录 | 用户询问小龙虾做了什么 | 展示巡群、捞 @、总结、提醒记录 |
| 日记 | 第一人称小龙虾日记 | 把工作经历写成有情绪的日记 |
| 卡通图 | 为日记生成小龙虾配图 | 优先真实生成，失败则模板降级 |
| 成就装扮 | 用户和小龙虾双向成就 | 解锁“第一次领养”“消息打捞员”等 |
| 分享卡片 | 生成日记分享卡片 | 包含标题、金句、配图、勋章、脱敏摘要 |
| 小龙虾评论 | 生成角色化评论回复 | 对好友评论生成回复草稿，默认预览 |
| 小龙虾空间 | 展示空间首页 | 包含动态、日记、勋章墙、评论区预览 |
| 进群权限 | 展示龙虾进群流程 | 用户邀请、群管理确认、能力范围说明 |

### 4.2 不做范围

| 不做项 | 原因 | Demo 替代方案 |
| --- | --- | --- |
| 真实 QQ 登录 | 初赛 Demo 不需要真实接入，接入成本高 | 使用模拟 QQ 用户和群聊 |
| 真实读取群聊 | 涉及权限、接口和隐私 | 使用可控模拟消息数据 |
| 真实发送消息 | 社交风险高，接口不可控 | 只展示发送前预览和用户确认 |
| 真实发布空间动态 | 非必要，且依赖 QQ 生态接口 | 做动态预览和空间预览 |
| 多用户实时共养 | 超出初赛核心闭环 | 仅展示长期方向说明 |
| 完整 QQ 秀系统 | 美术和系统复杂度过高 | 轻量小龙虾形象、勋章、挂饰 |
| 自动代表用户发言 | 权限风险高 | 默认评论和回复都只生成草稿 |

### 4.3 Demo 核心闭环

1. 用户进入 Demo，领养小龙虾。
2. 用户选择小龙虾性格、称呼和兴趣。
3. 用户选择授权的模拟 QQ 群和能力范围。
4. 小龙虾巡查模拟群聊，生成虾虾收件箱。
5. 用户查看待回复 @、任务、通知。
6. 用户选择某个群和时间段，让小龙虾总结。
7. 用户继续追问群聊内容。
8. 用户问“小龙虾今天做了什么”。
9. 小龙虾展示可解释工作记录。
10. 小龙虾生成第一人称日记。
11. 系统生成小龙虾卡通配图。
12. 系统发放用户成就和小龙虾成就。
13. 小龙虾佩戴新挂饰。
14. 系统生成日记分享卡片和动态预览。
15. 好友评论后，小龙虾生成角色化回复草稿。
16. 用户进入小龙虾空间，查看动态、日记、勋章墙。
17. Demo 展示龙虾进群权限预览。

## 6. AI 能力与数据流

### 6.1 输入数据

| 输入 | 来源 | 用途 |
| --- | --- | --- |
| 模拟群聊消息 | Demo 内置数据 | @ 收集、总结、问答、任务识别 |
| 用户性格 | 个性设置页 | 控制小龙虾提醒、日记、评论语气 |
| 兴趣画像 | 个性设置页生成或用户编辑 | 影响日记表达、装扮推荐、空间内容 |
| 授权范围 | 权限设置页 | 决定哪些群、哪些能力可被使用 |
| Demo 行为日志 | 用户在 Demo 中的操作 | 工作记录、成就触发、日记素材 |
| 好友评论 | 小龙虾评论页模拟数据 | 生成角色化回复 |
| 成就规则 | Demo 内置规则 | 触发用户成就和小龙虾成就 |

### 6.2 处理流程

1. 消息预处理：按群、时间、发送人、是否 @ 用户进行整理。
2. 事件抽取：识别 @、任务、截止时间、通知、情绪互动、冷场、链接、文件。
3. 重要性判断：按照是否 @ 用户、是否有时间、是否要求回复、是否来自关键人排序。
4. 收件箱生成：把事件归入待回复、任务提醒、群通知、情绪互动、可稍后。
5. 群聊总结：按照用户选择的群、时间段和总结模式生成结构化摘要。
6. 追问问答：基于当前群聊摘要和原始片段回答用户问题。
7. 工作记录生成：记录小龙虾巡了哪些群、捞到哪些消息、做了哪些总结。
8. 日记生成：结合工作记录、用户性格、小龙虾性格和成就生成第一人称日记。
9. 卡通图提示词生成：提取日记标题、情绪、挂饰和场景，生成配图提示词。
10. 成就判定：根据用户行为和小龙虾工作记录触发成就。
11. 评论生成：基于好友评论、小龙虾性格和分享内容生成回复草稿。
12. 空间动态生成：将日记、成就、挂饰和卡片聚合为空间动态。

| 输出 | 展示位置 | 要求 |
| --- | --- | --- |
| 虾虾收件箱 | 收件箱页、首页摘要 | 结构化、可处理、可解释 |
| 群聊总结 | 群聊总结页 | 有重点、有待办、有时间点 |
| 追问答案 | 群聊总结页 | 回答具体问题，必要时说明未找到 |
| 回复草稿 | 收件箱页、群聊总结页 | 默认不发送，用户可编辑 |

### 6.4 AI 原生性说明

Demo 必须突出 AI 原生能力，而不是静态页面拼接。

| AI 能力 | 如果没有 AI 会怎样 | Demo 证明方式 |
| --- | --- | --- |
| 群聊理解 | 只能按关键词搜索，无法理解上下文和任务 | 从模拟群聊中识别 ddl、分工、@、待回复 |

## 7. 权限、安全与隐私

### 7.1 权限原则

1. 默认只使用 Demo 内置模拟数据。
2. 用户必须按群授权，小龙虾不能默认看全部群。
3. 收集 @、总结群聊、生成草稿、写入日记素材需要分别说明。
4. 小龙虾评论和代发默认只预览，不自动发送。
5. 小龙虾做过什么必须可查看、可追问、可解释。
6. 分享日记和卡片前必须做脱敏提示。
7. 龙虾进群必须展示群管理确认和群成员可见公告。
8. 用户可以一键暂停小龙虾巡群。

### 7.2 权限分级

| 等级 | 能力 | Demo 默认 | 风险说明 |
| --- | --- | --- | --- |
| L0 | 使用用户手动导入或 Demo 模拟消息 | 开启 | 低风险 |
| L1 | 读取用户选择群聊的指定时间段 | 模拟开启 | 需用户明确授权 |
| L2 | 持续巡查指定群聊并提醒用户 | 仅演示 | 真实上线需可暂停 |
| L3 | 生成回复或评论草稿 | 开启 | 不代表用户发送 |
| L4 | 代用户发送消息 | 关闭 | 初赛不做真实发送 |
| L5 | 进群公开发言 | 仅权限预览 | 需用户和群管理确认 |

### 7.3 敏感内容处理

| 场景 | 风险 | Demo 策略 |
| --- | --- | --- |
| 群聊总结 | 可能暴露他人聊天内容 | 只展示模拟数据，并强调真实上线按授权范围处理 |
| 日记生成 | 可能写出群名、人名、隐私原文 | 默认使用关系称谓和摘要，不展示原文 |
| 分享卡片 | 可能把内部任务或聊天内容发出去 | 分享前脱敏，用户确认 |
| 小龙虾评论 | 可能越权代表用户表达 | 默认生成草稿，不自动发送 |
| 龙虾进群 | 群成员担心被 AI 读取 | 群管理确认，群成员可见公告，说明能力范围 |

### 7.4 必备信任文案

Demo 中需要出现以下类型文案：

1. “我只会查看你授权的群聊和时间范围。”
2. “我不会自动替你发消息。”
3. “这条日记已隐藏群名和人名。”
4. “这条评论只是草稿，发送前需要你确认。”
5. “龙虾进群需要群主或管理员同意。”
6. “你可以随时暂停我的巡群。”

## 12. 初赛说明文档可复用内容

### 12.1 产品概述

QQ 养虾是一款面向 QQ 社交场景的 AI 社交伙伴产品。用户可以领养一只属于自己的小龙虾，并授权它在指定 QQ 群聊中巡查消息、收集 @、总结群聊、回答追问和生成回复草稿。小龙虾会把自己帮助用户处理社交信息流的过程，转化为第一人称成长日记、双向成就、挂饰装扮、卡通分享卡片和小龙虾空间动态，让用户在 QQ 中获得实用的信息管理能力，也获得轻松、有趣、可传播的社交陪伴体验。

### 12.2 AI 原生说明

QQ 养虾的核心不是静态宠物养成，而是 AI Agent 对 QQ 社交信息流的理解和转化。AI 负责从群聊中识别 @、任务、截止时间和上下文，支持用户自然语言追问，并将小龙虾的工作记录转化为第一人称日记、评论、卡片文案和成就触发。如果去掉 AI，产品会退化为普通消息聚合或普通宠物装扮，核心价值会明显下降。

### 12.3 权限说明

QQ 养虾默认只在用户授权范围内工作。用户可以按群授权，可以查看小龙虾做过什么，也可以随时暂停巡群。小龙虾生成的回复、评论、日记和分享卡片默认由用户确认后才可发送或展示。龙虾进群需要群主或管理员确认，并向群成员展示能力范围和发言边界。

### 12.4 初赛 Demo 摘要

初赛 Demo 使用模拟 QQ 群聊数据，展示一个完整闭环：用户领养小龙虾，设置性格和兴趣，授权指定群聊，小龙虾巡群捞 @ 并生成收件箱；用户对群聊进行总结和追问；小龙虾展示工作记录，生成第一人称日记、卡通配图、双向成就、分享卡片、角色化评论和小龙虾空间；最后展示龙虾进群的权限预览。

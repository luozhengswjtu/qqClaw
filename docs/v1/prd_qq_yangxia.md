## 1. 产品一句话

QQ 养虾是一个属于用户自己的 AI 社交伙伴。它以“小龙虾”的身份陪伴用户使用 QQ，帮助用户巡群、收集 @、总结群聊、回答错过的消息，并把自己和用户共同经历的社交瞬间转化为第一人称日记、双向成就、装扮和可分享动态。

## 2. 核心判断

本产品不应被定义为“AI 写日记工具”，也不应只是“QQ 宠物复刻”。它的核心是：

1. 外层是有性格、有成长、有形象的小龙虾伙伴。
2. 内核是能处理 QQ 社交信息流的 AI Agent。
3. 情绪价值来自小龙虾第一人称叙事和成长反馈。
4. 实用价值来自 @ 收集、群聊总结、问答、提醒、回复草稿。
5. 社交传播来自日记卡片、勋章、挂饰、小龙虾动态和小龙虾空间。

用户需要感受到：这不是一个通用机器人，而是“我的虾”。

## 3. 背景与机会

### 3.1 QQ 场景特点

QQ 仍承载大量校园、兴趣、社群、游戏、班级、社团和项目协作场景。对学生用户而言，QQ 群往往不是单一聊天工具，而是信息流、关系链、身份展示和社交资产的集合。

典型问题包括：

1. 群聊消息量大，重要 @、任务、截止时间容易被淹没。
2. 用户想补课某段群聊，但不想翻 99+ 消息。
3. 用户希望维护社交关系，但缺少轻量、有趣、不尴尬的表达方式。
4. 传统 AI 助手偏工具，缺少长期陪伴感和角色记忆。
5. 传统宠物养成偏娱乐，缺少和真实 QQ 行为的连接。

### 3.2 赛题机会

赛题要求设计“官方 Agent”，强调主动感知场景、理解用户意图，并贯穿聊天、群聊、动态等核心流程。QQ 养虾可以同时命中：

1. Agent：小龙虾能巡群、总结、提醒、问答和生成回复草稿。
2. 主动感知：在用户授权后识别 @、待办、重要通知和群聊变化。
3. 情感连接：通过小龙虾第一人称日记，把工具行为转成关系记忆。
4. 社交新玩法：通过勋章、挂饰、日记卡片、小龙虾空间形成传播。
5. QQ 生态适配：群聊、动态、空间、装扮、身份展示均与 QQ 强相关。

## 4. 产品目标

### 4.1 用户目标

1. 不错过 QQ 群里和自己相关的重要消息。
2. 快速理解一段时间内群聊发生了什么。
3. 可以自然地询问小龙虾“你帮我做了什么”“我错过了什么”。
4. 通过小龙虾日记回看自己和 QQ 社交关系中的有趣瞬间。
5. 拥有一只可自定义、有成长、有记忆、有装扮的专属小龙虾。
6. 通过卡片、动态、勋章和评论进行轻量社交传播。

### 4.2 产品目标

1. 将 QQ 的高频社交行为转化为可理解、可行动、可沉淀的 AI 体验。
2. 让 AI Agent 从“工具助手”升级为“有角色关系的社交伙伴”。
3. 建立用户与小龙虾之间的长期养成关系，提高留存和分享意愿。
4. 在权限透明的前提下探索 AI 进入群聊、动态和空间的边界。

### 4.3 初赛 Demo 目标

初赛阶段不追求真实接入 QQ 数据，而是用模拟 QQ 群聊数据完成一个强闭环：

用户领养小龙虾 -> 设置性格与兴趣 -> 授权选择群聊 -> 小龙虾收集 @ 和重要消息 -> 用户进行群聊总结和问答 -> 小龙虾生成第一人称日记 -> 生成卡通配图和分享卡片 -> 双向发放成就 -> 小龙虾形象变化 -> 小龙虾评论预览 -> 小龙虾空间预览。

## 5. 非目标

初赛 Demo 阶段暂不做：

1. 真实 QQ 账号登录和真实群聊数据读取。
2. 真实发送 QQ 消息、空间动态或群评论。
3. 完整 QQ 秀级别的形象系统。
4. 多用户实时共养小龙虾和真实访客互动链路。
5. 完整商业化闭环。
6. 自动代表用户发言。

其中“自动代表用户发言”即使在长期版本中也应保持谨慎，默认必须用户确认。

## 6. 目标用户

### 6.1 核心用户

| 用户类型 | 描述 | 核心需求 |
| --- | --- | --- |
| 高校 QQ 重度用户 | 同时活跃在班级群、课程群、社团群、项目群、兴趣群 | 不错过 @、快速补课群聊、维护关系 |
| 社群活跃学生 | 经常参与活动、游戏、兴趣社群、频道讨论 | 需要总结信息、管理任务、表达身份 |
| 轻社交但怕错过用户 | 不常发言，但需要关注群消息和好友动态 | 希望被提醒，不想一直盯群 |
| 喜欢装扮和分享的用户 | 喜欢 QQ 秀、空间、头像、勋章、个性展示 | 需要可视化成长和可分享内容 |

### 6.2 次级用户

| 用户类型 | 描述 | 价值 |
| --- | --- | --- |
| 群主 / 管理员 | 管理班级群、社团群、项目群 | 可通过群聊共养虾提升信息整理效率 |
| 好友 / 群成员 | 看到用户分享的小龙虾日记、勋章和动态 | 产生互动、点赞、评论、访问小龙虾空间 |

## 7. 用户痛点

| 痛点 | 具体表现 | 机会 |
| --- | --- | --- |
| 重要消息被淹没 | QQ 群 99+ 后，@、通知、ddl 很难快速定位 | 小龙虾自动收集 @ 和重要消息 |
| 补课成本高 | 用户错过半天群聊，需要从头翻记录 | 群聊总结和问答 |
| AI 助手缺少关系感 | 普通助手只给结果，没有陪伴和成长 | 小龙虾第一人称日记与形象成长 |
| 社交表达有门槛 | 用户想分享生活但不想太正式或太暴露 | 小龙虾替用户以轻松方式表达 |
| 养成系统缺少真实连接 | 传统宠物成长常依赖点击、喂食、数值任务 | 基于真实 QQ 行为触发成长和成就 |
| Agent 主动性带来不安 | 用户担心 AI 偷看、乱发、打扰别人 | 透明权限、行为记录、确认后发送 |

## 8. 产品原则

1. 属于我：小龙虾的性格、兴趣、记忆、装扮和成长都应体现用户个人特征。
2. 有用先行：小龙虾必须能解决真实 QQ 信息流问题，而不是只做装饰。
3. 主动但可控：可以主动发现问题，但不能越权发言或暴露隐私。
4. 情绪化表达：日记、提醒和评论要有角色感，避免工具化报告。
5. 权限透明：用户和群成员都应知道小龙虾看到了什么、做了什么、能做什么。
6. 可追问：用户随时可以问小龙虾“你刚才做了什么”“为什么提醒我”。
7. 可传播：核心产物需要天然适合分享到 QQ 聊天、群聊、空间和动态。

## 10. 核心概念

### 10.1 小龙虾

小龙虾是用户在 QQ 生态中的 AI 社交伙伴，具备：

1. 性格：由用户选择或微调，例如温柔、毒舌、热血、社恐、秘书型。
2. 兴趣：根据用户画像生成，可被用户修改，例如游戏、AI、校园活动、二次元、考研、实习。
3. 记忆：记录与用户相关的公开成就、用户确认保留的偏好和小龙虾经历。
4. 能力：巡群、收集 @、总结、问答、提醒、生成回复草稿、写日记、生成卡片。
5. 形象：可佩戴勋章、挂饰、壳色、贴纸、空间装饰。

### 10.2 用户和小龙虾的双向成就

成就不是只奖励用户，也奖励小龙虾。两者共同构成关系成长。

| 类型 | 用户侧成就 | 小龙虾侧成就 |
| --- | --- | --- |
| 初次使用 | 第一次领养小龙虾 | 第一次认主 |
| 巡群 | 第一次选择授权群聊 | 第一次出门巡群 |
| @ 收集 | 第一次查看虾虾收件箱 | 第一次捞到 @ |
| 群聊总结 | 第一次总结群聊 | 第一次捞起 99+ 消息 |
| 回复草稿 | 第一次采纳回复建议 | 第一次替主人接话 |
| 日记 | 第一次查看小龙虾日记 | 第一次写下今天 |
| 分享 | 第一次分享日记卡片 | 第一次出门营业 |
| 进群 | 第一次带虾进群 | 第一次戴上群工牌 |

### 10.3 隐藏成就

隐藏成就用于刺激探索，但不应变成机械刷任务。建议以情境触发为主。

| 隐藏成就 | 触发示例 | 奖励示例 |
| --- | --- | --- |
| 冷场终结者 | 用户在冷场群聊中发起话题并获得回应 | 热锅破冰钳 |
| 深夜灯塔 | 用户深夜回应好友负面情绪 | 夜航虾灯 |
| 老友重联 | 用户与长时间未联系好友重新互动 | 回声贝壳 |
| 表情包外交官 | 用户用表情包化解尴尬或带动互动 | 表情包小背包 |
| 潜水但在乎 | 用户低发言但持续关注重要群消息 | 观察员潜水镜 |
| 生日救场 | 小龙虾提醒用户好友生日并生成祝福草稿 | 祝福彩带 |

### 10.4 小龙虾日记

日记不是用户操作日志，而是小龙虾第一人称成长记录。它应包含：

1. 今日具体事件。
2. 小龙虾的观察和情绪。
3. 用户与小龙虾的关系互动。
4. 小龙虾做了什么实事。
5. 成长、成就或装扮变化。
6. 适合分享的一句话金句。

### 10.5 日记卡片

日记卡片是传播载体，包含：

1. 小龙虾当天形象。
2. 今日标题。
3. 一句日记金句。
4. 今日获得的勋章或挂饰。
5. 简短日记摘录。
6. 用户与小龙虾关系称号。
7. 分享入口。

初赛 Demo 可以生成静态卡片，不需要真实发到 QQ。

### 10.6 龙虾进群

龙虾进群是 QQ 原生场景的重要能力，但必须重视权限。

分为三种模式：

| 模式 | 可见范围 | 说明 |
| --- | --- | --- |
| 私人随身虾 | 仅用户可见 | 小龙虾可在用户授权范围内巡群、收集 @、总结和提醒 |
| 派驻进群虾 | 群成员可见 | 用户邀请自己的小龙虾进群，需群主或管理员同意 |
| 群聊共养虾 | 群成员共同拥有 | 群维度的小龙虾，负责群总结、群成就和群日记 |

初赛 Demo 重点展示“私人随身虾”和“派驻进群虾”的概念，不做多人协作。

## 11. 核心用户流程

### 11.1 领养与初始化

1. 用户进入 QQ 养虾入口。
2. 系统展示小龙虾定位：帮你巡群、捞 @、写日记、长成你的社交伙伴。
3. 用户选择小龙虾性格。
4. 系统根据用户画像生成兴趣爱好建议。
5. 用户确认或修改兴趣。
6. 用户选择授权的群聊和数据范围。
7. 小龙虾生成初始形象和第一条认主日记。

### 11.2 虾虾收件箱

1. 用户授权小龙虾巡查指定群聊。
2. 小龙虾收集时间段内 @ 用户、回复用户、重要通知、任务信息。
3. 系统按类型和紧急程度分类。
4. 用户进入虾虾收件箱查看。
5. 用户可点击某条消息查看上下文摘要。
6. 用户可让小龙虾生成回复草稿。
7. 用户确认后自行发送或由小龙虾代发。

### 11.3 群聊总结与问答

1. 用户选择一个 QQ 群。
2. 用户选择时间段，例如最近 2 小时、今天、昨天晚上、自定义时间。
3. 用户选择总结类型：全部总结、只看和我有关、任务总结、争议总结、补课模式。
4. 小龙虾生成结构化摘要。
5. 用户继续追问，例如“最后谁负责”“有没有人 cue 我”“我需要回什么”。
6. 小龙虾回答，并保留引用来源或上下文依据。

### 11.4 询问小龙虾做了什么

1. 用户问：“你今天帮我做了什么？”
2. 小龙虾展示工作记录：
   - 巡了哪些群。
   - 收集了哪些 @。
   - 生成了哪些总结。
   - 提醒了哪些事项。
   - 是否生成过回复草稿。
   - 是否发过消息，若发过需说明由谁确认。
3. 用户可追问“为什么提醒我这个”“这条消息原文在哪”。

### 11.5 小龙虾日记

1. 每天或每次完成关键任务后，小龙虾生成日记草稿。
2. 日记素材来自：
   - 用户授权行为摘要。
   - 小龙虾工作记录。
   - 今日解锁成就。
   - 用户和小龙虾互动。
3. 用户可选择语气：温柔、吐槽、热血、简短、故事化。
4. 用户可编辑或隐藏敏感内容。
5. 用户确认保存。
6. 系统生成日记卡片和可分享动态。

### 11.6 成就、装扮与分享

1. 系统根据用户行为和小龙虾工作记录判断成就。
2. 同时发放用户侧成就和小龙虾侧成就。
3. 小龙虾获得对应挂饰、勋章或形象变化。
4. 用户可在小龙虾形象页佩戴。
5. 用户可生成日记卡片、勋章卡片或小龙虾动态。
6. 好友可点赞、评论、访问小龙虾空间。
7. 小龙虾可在评论区以角色口吻回复，默认需用户确认。

### 11.7 龙虾进群

1. 用户在群聊中选择“邀请我的小龙虾进群”。
2. 系统展示小龙虾能力和数据范围。
3. 群主或管理员确认。
4. 群成员可看到小龙虾身份说明和权限。
5. 小龙虾默认只在被 @、用户确认或群规则允许时发言。
6. 小龙虾可获得群专属成就和装扮，例如群工牌。

### 12.2 P1：可作为说明文档扩展

## 13. 关键页面设计

### 13.1 小龙虾首页

页面目标：让用户一眼看到“我的虾今天怎么样、帮我做了什么、我该处理什么”。

核心元素：

1. 小龙虾形象区：展示当前装扮、勋章、状态。
2. 今日一句话：小龙虾对用户说的话。
3. 今日待处理：待回复 @、重要任务、提醒。
4. 今日工作记录入口：查看小龙虾做了什么。
5. 今日日记入口：生成或查看日记。
6. 今日成就：新获得的用户成就和小龙虾成就。

### 13.2 虾虾收件箱

页面目标：解决用户错过 @ 和重要群消息的问题。

核心元素：

1. 分类标签：待回复、任务、通知、情绪互动、可稍后。
2. 消息卡片：来源群、发送人、摘要、时间、重要程度。
3. 上下文按钮：查看前后消息摘要。
4. 回复草稿按钮：让小龙虾生成回复建议。
5. 标记已处理。

### 13.3 群聊总结页

页面目标：让用户快速补课某段群聊。

核心元素：

1. 群聊选择器。
2. 时间范围选择器。
3. 总结模式选择：全部、只看和我有关、任务、补课、争议。
4. 结构化总结：
   - 发生了什么。
   - 和我有关的内容。
   - 待办事项。
   - 重要时间点。
   - 可回复建议。
5. 追问输入框。

### 13.4 小龙虾日记页

页面目标：把实用行为转成有情绪价值的关系内容。

核心元素：

1. 日记标题。
2. 第一人称正文。
3. 今日事件来源提示。
4. 今日获得成就。
5. 敏感内容检查和隐藏开关。
6. 生成卡片。
7. 分享到聊天、群聊、空间、小龙虾空间的预览入口。

### 13.5 成就与装扮页

页面目标：沉淀长期成长和用户探索欲。

核心元素：

1. 用户成就墙。
2. 小龙虾成就墙。
3. 隐藏成就线索。
4. 小龙虾挂饰和勋章装备区。
5. 今日获得来源说明。
6. 分享成就卡片。

### 13.6 权限与记忆页

页面目标：降低用户对 AI 主动感知的担忧。

核心元素：

1. 已授权群聊列表。
2. 每个群的可读取范围。
3. 小龙虾是否可提醒、是否可生成草稿、是否可发言。
4. 行为记录。
5. 记忆管理：查看、编辑、删除。
6. 一键暂停巡群。
7. 隐私说明。

## 14. AI 能力设计

### 14.1 AI 能力清单

| 能力 | 用途 |
| --- | --- |
| 消息理解 | 识别 @、任务、通知、情绪、关系互动 |
| 重要性判断 | 判断哪些消息需要用户关注 |
| 群聊总结 | 将一段群聊转为结构化摘要 |
| 基于上下文问答 | 回答用户对群聊的追问 |
| 回复草稿生成 | 根据上下文生成合适语气的回复建议 |
| 用户画像生成 | 根据授权行为生成兴趣、社交风格建议 |
| 小龙虾人格生成 | 根据用户选择的性格输出一致口吻 |
| 日记生成 | 生成小龙虾第一人称成长日记 |
| 成就判定 | 根据行为事件触发用户和小龙虾成就 |
| 卡片文案生成 | 将日记压缩为适合分享的标题、金句、摘录 |
| 卡通图生成 | 生成简单小龙虾插图或卡片视觉元素 |

### 14.2 AI 工作流

1. 用户授权群聊和时间范围。
2. 系统获取或模拟消息数据。
3. 消息预处理：去重、分段、脱敏、按时间排序。
4. 事件抽取：@、任务、通知、情绪互动、冷场、回应、截止时间。
5. 重要性评分：判断需要提醒、可稍后、可忽略。
6. 输出结构化结果：收件箱、总结、待办、回复建议。
7. 生成小龙虾工作记录。
8. 基于工作记录和用户行为生成日记。
9. 判断成就和装扮变化。
10. 生成日记卡片和分享文案。

### 14.3 AI 原生性说明

如果去掉 AI，本产品会退化为普通消息聚合、普通宠物养成或手工日记工具，核心价值会明显下降。

AI 原生性体现在：

1. 只有 AI 能从非结构化群聊中理解 @、任务、情绪和上下文。
2. 只有 AI 能支持用户自然语言追问群聊内容。
3. 只有 AI 能把同一组行为转化为不同性格的小龙虾表达。
4. 只有 AI 能生成有具体事件、有情绪弧线、有关系感的第一人称日记。
5. 只有 AI 能将用户画像、兴趣、社交行为和视觉卡片结合为个性化产物。

## 15. 权限与安全设计

### 15.1 权限原则

1. 默认关闭，用户主动授权。
2. 按群授权，不默认读取全部群聊。
3. 按能力授权，例如总结、提醒、回复草稿、发言分开控制。
4. 用户可随时暂停、撤销、删除。
5. 小龙虾每次主动提醒都应能解释来源。
6. 默认不自动发言，用户确认后才发送。
7. 龙虾进群需群主或管理员同意，并向群成员展示能力范围。

### 15.2 权限分级

| 权限等级 | 能力 | 风险 | 默认策略 |
| --- | --- | --- | --- |
| L0 | 仅使用用户手动导入的消息 | 低 | Demo 默认 |
| L1 | 读取用户选择群聊的指定时间段 | 中 | 用户主动授权 |
| L2 | 持续巡查指定群聊并提醒用户 | 中高 | 明示开启，可暂停 |
| L3 | 生成回复草稿 | 中 | 默认允许，但不发送 |
| L4 | 代用户发送消息 | 高 | 默认关闭，单次确认 |
| L5 | 进群公开发言 | 高 | 需用户和群管理确认 |

### 15.3 龙虾进群权限

派驻进群虾需要展示：

1. 这是谁的小龙虾。
2. 它能做什么。
3. 它能看到哪些内容。
4. 它什么时候会说话。
5. 它是否会把群内容写入用户日记。
6. 群成员如何查看、投诉或要求移除。

推荐默认文案：

> 小龙虾已进入本群。它是用户的 AI 社交伙伴，可在授权范围内帮助用户总结群聊、收集 @ 和生成回复草稿。它不会默认代表用户发言，也不会在未确认的情况下发布群内容。

### 15.4 隐私风险控制

| 风险 | 控制方式 |
| --- | --- |
| 用户觉得被监控 | 权限页、工作记录、可追问来源 |
| 群成员担心内容外泄 | 进群明示、群管理同意、群成员可见权限 |
| 日记暴露隐私 | 敏感内容检查、用户确认后分享 |
| AI 误判重要性 | 保留来源、允许用户反馈“不是重点” |
| AI 乱发言 | 默认只生成草稿，发送需确认 |
| 角色表达冒犯用户 | 性格可调，提供语气边界和安全过滤 |

## 16. 内容与人格设计

### 16.1 小龙虾语气原则

1. 像伙伴，不像客服。
2. 有趣但不油腻。
3. 可以吐槽，但不伤害用户。
4. 可以拟人，但不能伪装成真人。
5. 对隐私内容保持克制。
6. 实用提醒要简洁，日记表达可以更有文学感。

### 16.2 性格选项

| 性格 | 表达特点 | 适合用户 |
| --- | --- | --- |
| 温柔陪伴型 | 细腻、鼓励、情绪支持 | 压力大、重视陪伴 |
| 毒舌但靠谱 | 轻微吐槽、关键时刻可靠 | 喜欢玩梗、接受调侃 |
| 热血队友型 | 积极、行动感强 | 项目协作、竞赛用户 |
| 社恐观察型 | 低调、细节观察、内心戏 | 潜水用户 |
| 高冷秘书型 | 简洁、效率、少废话 | 工具导向用户 |
| 搞笑整活型 | 梗多、活泼、适合分享 | 社交活跃用户 |

### 16.3 提醒文案示例

| 场景 | 文案 |
| --- | --- |
| 捞到 @ | 我刚从 99+ 里捞到 3 条和你有关的消息，其中 1 条需要今天回复。 |
| 发现任务 | 这条消息有截止时间，我已经给它夹上小红旗了。 |
| 总结完成 | 我把刚才两小时的群聊捞完了，重点是分工、ddl 和一个没人认领的表。 |
| 生成草稿 | 我写了一版不太生硬的回复，你看看要不要发。 |
| 工作记录 | 今天我巡了 3 个群，捞到 7 条 @，生成了 2 次总结，没有替你自动发送任何消息。 |

### 16.4 日记示例

标题：我第一次从 99+ 里捞到了主人

正文：

今天我第一次正式巡群。

课设群的消息像一锅开水，咕嘟咕嘟冒到 99+。我一开始还有点慌，钳子都不知道先夹哪条。后来我发现，有一条 @ 主人的消息被压在中间：明晚 20:00 前要交分工表。

我把它捞出来，夹上小红旗，放进了收件箱。

主人点开的时候只说了一句：“还好你看到了。”

我表面很镇定，其实虾壳都亮了一下。原来我不是只会写日记的虾，我真的能帮主人从一堆消息里找到重要的东西。

今天我获得了第一枚勋章：消息打捞员。

明天我还想再巡一圈。不是因为我爱工作，是因为我有点喜欢这种“差点错过，但被我夹住了”的感觉。

| --- | --- |
| 好友夸小龙虾会写 | 别夸太大声，我的钳子会翘起来。 |

## 18. 分享与传播设计

### 18.1 可分享内容

| 内容 | 分享价值 |
| --- | --- |
| 日记卡片 | 有故事、有情绪，适合空间和好友 |
| 成就卡片 | 展示成长和个性 |

### 18.2 分享前检查

任何包含群聊或好友信息的内容，分享前都应进行：

1. 人名脱敏或替换为关系称谓。
2. 群名可隐藏。
3. 原文默认不展示，只展示摘要。
4. 用户确认。
5. 高敏感内容提示。

### 18.3 小龙虾空间

小龙虾空间是长期展示页，可包含：

1. 小龙虾形象。
2. 日记时间线。
3. 勋章墙。
4. 挂饰展示。
5. 最近动态。
6. 好友留言和点赞。
7. 小龙虾评论。

初赛 Demo 可做空间预览，不做完整社交链路。

## 21. 风险与应对

| 风险 | 描述 | 应对 |
| --- | --- | --- |
| 被认为是监控工具 | 小龙虾主动巡群可能让用户或群成员不安 | 默认关闭、按群授权、工作记录、可追问来源 |
| 群成员隐私担忧 | 用户的小龙虾进入群后，群成员担心内容被带走 | 群管理同意、群成员可见说明、分享前脱敏 |
| AI 误判重要消息 | 可能漏掉重要 @ 或误报普通消息 | 展示来源、允许反馈、保留人工确认 |
| 自动发言带来社交风险 | AI 语气不合适或误发 | 默认只生成草稿，发送需确认 |
| 日记过度暴露用户生活 | 第一人称日记可能隐含真实关系 | 敏感内容检查、用户编辑、默认私密 |
| 养成系统变成刷任务 | 用户为了成就机械操作 | 强调情境型隐藏成就和真实社交价值 |
| 角色表达过度 | 小龙虾太吵、太油、太像客服 | 性格可选、语气可调、提醒频率可控 |
| Demo 落地过重 | 真实 QQ 接入难度高 | 初赛使用模拟数据，重点展示产品闭环和 AI 价值 |

## 22. 竞品与差异化

| 类型 | 代表 | 差异 |
| --- | --- | --- |
| 普通 AI 助手 | 通用聊天助手、总结工具 | 缺少 QQ 场景、长期关系和社交传播 |
| 群聊机器人 | 通知机器人、群管理机器人 | 偏工具和管理，不是用户专属伙伴 |
| 虚拟宠物 | QQ 宠物、养成类应用 | 有陪伴但缺少对真实社交信息流的处理 |
| 日记工具 | AI 日记、情绪记录 | 偏个人记录，缺少 Agent 行为和 QQ 社交资产 |
| 成就系统 | 游戏勋章、社区徽章 | 多为行为奖励，缺少小龙虾第一人称叙事和装扮反馈 |

QQ 养虾的差异化是：把 QQ 社交行为、AI 实用能力、宠物养成、日记叙事和社交展示连接成一个闭环。

## 25. 初赛说明文档可用摘要

产品概述：

QQ 养虾是一款面向 QQ 社交场景的 AI 社交伙伴产品。用户可以领养一只属于自己的小龙虾，并授权它在指定 QQ 群聊中巡查消息、收集 @、总结群聊、回答追问和生成回复草稿。小龙虾会把自己帮助用户处理社交信息流的过程，转化为第一人称成长日记、双向成就、挂饰装扮和可分享卡片，让用户在 QQ 中获得实用的信息管理能力，也获得轻松、有趣、可传播的社交陪伴体验。

核心价值：

1. 对用户：减少错过重要群消息的焦虑，降低补课群聊成本。
2. 对 QQ：把 AI Agent 深度嵌入聊天、群聊、动态和空间流程。
3. 对社交：把用户行为转化为可分享、可展示、可互动的社交资产。
4. 对 AI：从通用工具转变为有性格、有记忆、有成长的智能伙伴。

核心闭环：

用户授权群聊 -> 小龙虾巡群捞 @ -> 群聊总结与问答 -> 工作记录可追问 -> 小龙虾第一人称日记 -> 用户和小龙虾双向成就 -> 形象装扮变化 -> 生成分享卡片 / 小龙虾动态。

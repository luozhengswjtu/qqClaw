# 可直接发给其他 AI 的实施提示词 v2：QQ 主界面里的小龙虾 Agent Demo

下面这份提示词用于让其他 AI 编程助手实现或重构一个 **QQ 风格的交互式静态产品 Demo v2**。

目标不是做一个“QQ 养虾独立产品后台”，而是做一个：

> **QQ 主界面里的小龙虾 Agent 层**

也就是：用户打开 QQ，就能自然看到小龙虾入口；用户进入群聊时，小龙虾以提示卡、抽屉、私聊会话、日记弹卡、资料卡和空间动态的方式出现；不做真实后端，但页面、流程、状态变化都要像真实产品。

---

## 一、你要实现的东西

你是一名资深前端产品工程师和 UI 工程师。请在当前项目中实现一个 **QQ 风格的交互式静态产品 Demo**，产品名为 **QQ 养虾 / 我的小龙虾**。

这个 Demo 用于腾讯 PCG 校园 AI 产品创意大赛初赛，必须满足以下要求：

1. 不做介绍页，不做营销页，不做 hero 页。
2. 第一屏就是 QQ 主界面，而不是功能导航页。
3. 不接真实后端，不接真实 QQ 登录，不接真实群聊数据。
4. 允许使用本地 mock 数据和 mock AI adapter。
5. 页面、流程、状态变化、消息结构、卡片反馈，都要像一个真实产品。
6. 产品主线必须体现：
   - QQ 主界面
   - 小龙虾自然入口
   - 聊天式交互
   - 卡片式结果
   - 日记弹卡
   - 成就和挂饰
   - 小龙虾资料卡
   - 小龙虾自己的 QQ 空间
   - 群设置中的进群权限预览

这个 Demo 的最终观感应该是：

> 用户正在用 QQ，小龙虾像一个原生的智能伙伴自然出现在聊天流里；它既能帮用户捞重点消息、总结群聊、给回复建议，也能在关键时刻产生日记、成就和空间动态。

兴趣系统首版范围必须收敛：

- 第一版主打音乐兴趣纵深，后续围绕模拟 QQ 音乐授权、歌手、歌曲、曲风和演出提醒展开。
- 兴趣社群推荐保留为通用能力，但 Demo 只用羽毛球公开社群资料 mock 卡做样例。
- 不要在第一版铺满所有兴趣类型，不做复杂多兴趣调度，不做完整推荐算法。
- Demo 初始兴趣画像必须表达为“用户已经选择、表达、授权或设置过”的模拟状态，并在卡片里说明“为什么知道”和“来自哪里”。

## 三、总原则

### 1. 这是 QQ 主界面，不是独立产品后台

必须让用户打开 Demo 后，首先看到：

- 消息列表
- 群聊窗口
- 输入框
- 群资料区
- 小龙虾提示条 / 小龙虾私聊入口

而不是：

- 首页
- 功能面板
- 成就页
- 日记页
- 空间页
- 工作记录页

### 2. 聊天式交互是第一主交互

用户和小龙虾的主要关系，是“像和好友聊天一样问它”，而不是点来点去切页面。

### 3. 卡片式结果是第二主交互

小龙虾的输出要优先以卡片呈现：

#### 轻量文本卡片

这些不需要重美术：

- 群聊总结卡片（可切换多个群，含相关 @ 和摘要）
- 回复草稿卡片
- 工作记录卡片

#### 强视觉卡片

这些需要更好看的视觉：

- 日记弹卡
- 成就卡片
- 分享卡片
- 小龙虾空间动态卡

### 4. 先结果，后详情

用户看到卡片先完成任务；只有需要深挖时，才进入资料卡、空间页、权限页、群设置小钳模块。

### 5. 不做真实后端，但要像真实产品

虽然是静态 Demo，但必须有：

- 状态变化
- 交互闭环
- mock 数据
- mock AI 输出
- 权限边界
- 触发时机
- 反馈逻辑

不能只是漂亮的静态拼图。

## 六、必须实现的产品结构

### A. QQ 主界面壳子

这是第一屏，必须先实现。

布局建议：

```text
左侧：会话列表
中间：聊天主区域
右侧：群资料 / 文件 / 成员信息
顶部：群标题、状态、轻量操作
底部：输入框
悬浮 / 附加层：小龙虾提示条、小龙虾抽屉、小龙虾私聊层、日记弹卡、成就 toast
```

至少包含这些会话：

1. 软件工程课设群
2. AI 产品创意赛群
3. 班级通知群
4. 游戏开黑群
5. 小钳（小龙虾专属会话）

### B. 小龙虾自然入口

必须同时体现这几种入口：

1. **打开 QQ 时**
   - 消息列表里有 “小钳” 会话
   - 或显示 “我帮你捞到重点了” 小红点提示

2. **进入课设群时**
   - 群聊顶部出现提示条：
     `小钳帮你捞到 3 条重点，其中 1 条需要今天回复。`

3. **输入框旁**
   - 有小龙虾按钮，可快速拉起总结 / 问答 / 回复建议

4. **点击头像**
   - 进入小龙虾资料卡

5. **群设置中**
   - 有“小钳设置 / 邀请我的小龙虾进群”入口

### C. 小龙虾聊天式交互层

必须有一个类似“和好友聊天”的小龙虾私聊窗。

支持这些典型输入：

1. 今天你帮我做了什么？
2. 刚才群里发生了什么？
3. 这张总结卡里有哪些和我有关的消息？
4. 我需要回什么？
5. 帮我写一句轻松一点的回复。
6. 你今天写日记了吗？

这个会话窗里，小龙虾不是纯文本回复，而是插入卡片。

## 七、必须实现的卡片体系

### 1. 群聊总结卡片

这是轻量文本卡片，用于承载一个或多个授权群的总结结果；每个群用切换入口展示。@ 消息不是独立卡片，而是对应群总结里的可选提醒信号，没有相关 @ 时不显示 @ 区域。

结构必须包含：

- 标题
- 群名
- 摘要
- 相关 @
- 时间
- 发送人
- 为什么重要
- 操作按钮

示例：

```text
标题：群聊总结
群：软件工程课设群
摘要：刚才主要在讨论模块分工和分工表提交
时间：10:17
发送人：组长
相关 @：@你确认是否负责接口文档
原因：与你直接相关，且影响明晚的分工表提交
操作：看上下文 / 写回复草稿 / 总结本群
```

### 2. 群聊总结摘要区

这是群聊总结卡片内的摘要区，不再作为独立于 @ 的第二张卡片。

结构必须包含：

- 本段主要讨论什么
- 和我有关
- 待办事项
- 重要时间点
- 建议回复

示例：

```text
课设群刚才主要在讨论模块分工和分工表提交。
和你有关的是接口文档任务，组长在 10:17 @ 你确认是否负责。
明晚 20:00 前需要提交分工表。
建议你回复：“可以，我负责接口文档。今晚我补一版权限说明，明天一起合进分工表。”
```

### 3. 回复草稿卡片

也是轻量文本卡片。

结构：

- 建议回复
- 语气
- 来源上下文
- 操作按钮

必须显式写：

```text
这条回复只是草稿，发送前需要你确认。
```

### 4. 工作记录卡片

这是文本卡片，不是页面。

当用户问“小龙虾今天做了什么”时返回。

示例内容：

```text
队长，我今天巡了 3 个你授权的群，捞到 4 条和你有关的消息。
其中 2 条需要回复，1 条是明晚 20:00 的 ddl，1 条是班级通知。
我生成了 1 次课设群总结、1 条回复草稿和 1 篇日记。
回复草稿已准备好，可以复制后跳回原群使用。
```

必须支持一个“查看详情”展开，但不能跳到独立工作记录页。

### 5. 日记弹卡

这是强视觉卡片，不是独立页面。

它必须在一个“时机”下自动或半自动出现，例如：

- 用户刚处理完重点消息
- 小龙虾第一次成功捞到重要 @

日记弹卡必须包含：

1. 配图
2. 标题
3. 一句金句
4. 日记摘要
5. 当前成就提示
6. 操作：
   - 分享
   - 存入小虾空间
   - 查看成就
   - 收起

重要限制：

1. 不支持反复重新生成整篇日记。
2. 可以支持：
   - 切换全文 / 摘要
   - 隐藏敏感信息
   - 更换配图
   - 存入空间

### 6. 成就 toast / 成就卡片

成就不做一级页面入口。

必须在关键节点出现即时反馈：

```text
小钳获得「消息打捞员」
```

同时资料卡里能看到最近成就。

### 7. 分享卡片

分享卡片的主要作用不是承接评论，而是：

1. 对外传播
2. 让好友感兴趣
3. 把好友引流到“小龙虾空间”或产品体验入口

所以分享卡片上应有：

- 日记标题
- 金句
- 小龙虾配图
- 成就 / 挂饰
- 已脱敏说明
- “查看我的小虾空间”或类似 CTA

### 8. 小龙虾空间动态卡

小龙虾空间里的动态要像 QQ 空间动态。

注意：

1. 这个空间的动态发布者是小龙虾，不是人类用户。
2. 人类用户可以浏览、点赞、评论、分享。
3. 人类用户不能直接替它在这个空间发动态。
4. 好友拥有的小龙虾也可以来互动或评论。

## 八、必须实现的深度承接视图

### 1. 小龙虾资料卡

8. 去权限设置

### 2. 小龙虾空间页

这是一个完整深度页，但不要喧宾夺主。

必须表现成：

> 小龙虾自己的 QQ 空间

必须包含：

1. 空间头图
2. 小龙虾形象
3. 由小龙虾发布的今日动态
4. 日记时间线
5. 勋章墙

7. 评论区
8. 小龙虾回复草稿入口

评论逻辑必须改成：

```text
评论发生在小龙虾空间动态下面
不是发生在分享卡片本身上
```

### 3. 权限设置页

必须包含：

1. 按群多选授权
2. 能力开关：
   - 整理并总结授权群聊
   - 生成回复草稿
   - 提醒用户

4. 一键暂停

必须显式出现这些文案：

1. 我只看你授权的群。
2. 这条回复只是草稿。

### 4. 群设置里的小钳模块

必须展示“邀请我的小龙虾进群”的预览能力。

需要包含：

1. 邀请按钮
2. 群管理确认状态
3. 进群后能力列表
4. 进群限制说明
5. 群成员可见公告预览

公告文案建议：

```text
小龙虾已进入本群。它是用户的 AI 社交伙伴，可在授权范围内整理群聊重点、识别 @ 信号和生成回复草稿。回复草稿需要用户复制或确认后再使用。
```

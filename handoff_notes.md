# QQ 养虾 v2 新线程接手说明

更新时间：2026-04-30

## 快速结论

当前任务不是继续修改历史 Demo，也不是一次性生成完整 Demo。

当前正确方向是：

> 分阶段从零实现 `apps/qq-lobster-v2`，先搭 QQ 主界面和小龙虾首次出现，再做认养、小龙虾交互界面、新手任务、权限、核心卡片、隐藏日记、小龙虾空间。

## 当前关键用户要求

1. 从零实现 v2，不依赖历史 Demo 目录、旧实现代码或旧信息架构。
2. 用户会按阶段验收并反馈。
3. 需要动态调整，不要一次性做完整 Demo。
4. 第一屏是 QQ 主界面。
5. 小龙虾首次出现应像用户平常使用 QQ 时自然触发。
6. 认养流程包括性格、称呼、兴趣初始化。
7. 认养后进入小龙虾主交互界面。
8. 小龙虾主交互界面参考用户草图：
   - 左侧聊天窗口索引
   - 中间小龙虾聊天窗口
   - 右侧小龙虾直观可视化界面和引导任务
9. 功能需要一步步引导，不要一次性暴露。
10. 日记是隐藏惊喜，在用户体验若干能力后触发。
11. 第一次日记看完后，才可以在交互界面中查看历史日记。
12. @ 信息和群聊总结需要支持一键跳转来源。
13. 小龙虾空间是龙虾自己的 QQ 空间，动态由龙虾发布。
14. 用户可以使用 `gpt-image-2` 协助生成图片资产，资产需求要及时拆给用户。
15. 小龙虾形象偏 Q 版可爱、非拟人，提高用户接触意愿。
16. 第一阶段只做 QQ 主界面、小龙虾首次自然出现、认养入口和认养流程，先验证自然出现感。
17. 讨论形成的新信息要及时更新到文档，方便新线程接手。

## 不要做的事

1. 不要以历史 Demo 的功能导航或多页面工作台为基础。
2. 不要把工作记录做成一级页面。
3. 不要把日记做成“点击生成”页面。
4. 不要让用户一开始就看到所有功能。
5. 不要默认打开群聊并展示已经总结好的结果。
6. 不要忽略权限引导。
7. 不要真实接后端或真实 QQ。
8. 不要把小龙虾做成人形拟人角色。

## 建议下一步

执行阶段 0 和阶段 1：

1. 新建 `apps/qq-lobster-v2`。
2. 搭建 Vite + React + TypeScript + Tailwind。
3. 创建基础状态层、mock 数据和 AI adapter 空壳。
4. 实现 QQ 主界面壳子。
5. 实现小龙虾首次出现动画和认养入口。

阶段 1 完成后停止，给用户验收。

## 每阶段完成后应更新

1. `implementation_state.md`
2. `stage_plan.md`
3. 如果有关键决策变化，更新本文件。

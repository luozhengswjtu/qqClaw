# QQ 养虾 v2 实施状态

更新时间：2026-05-02

## 当前结论

项目方向已从 v1 的“QQ 养虾独立产品工作台”调整为 v2 的“QQ 主界面里的小龙虾 Agent 层”。

核心产品体验：

1. 用户像往常一样打开 QQ。
2. 在合适时机看到小龙虾动画入口。
3. 点击后完成小龙虾认养。
4. 进入小龙虾主交互界面。
5. 在小龙虾聊天界面中通过打卡一步步体验能力：对话、授权、群聊感知、来源跳转、回复草稿、工作记录、空间动态、评论回复。
6. 日记作为隐藏惊喜，在用户完成若干操作后触发。
7. 小龙虾空间是小龙虾自己的 QQ 空间，动态由小龙虾发布。

## 硬约束

1. v2 从零实现，不依赖任何历史 Demo 目录、旧实现代码或旧信息架构。
2. 历史实现目录若仍存在，应视为可删除的废弃产物，不作为代码或设计基线。
3. 新实现应另起目录，建议为 `apps/qq-lobster-v2`。
4. 第一屏必须是 QQ 主界面，不是产品首页、功能导航或营销页。
5. 小龙虾首次出现应是 QQ 使用过程中的自然触发，不是用户主动打开功能中心。
6. 日记第一次出现必须是隐藏惊喜，不是用户点击“生成日记”。
7. 工作记录、@ 消息、群聊总结、回复草稿优先以聊天窗口里的文本卡片呈现。
8. 日记、成就、分享、空间动态优先以强视觉卡片呈现。
9. 所有 AI 输出必须通过 adapter 层，禁止直接散落在页面 JSX 中。

## 新增讨论结论：OpenClaw Agent 底座

用户确认：阶段 1.5 已完成。OpenClaw 不只是数据库和 AI adapter，而应作为可持续演进的 Agent 底座。

当前共识：

1. OpenClaw 的目标是最小真实数据流底座，不是完整后端平台。
2. 真实 AI 第一版优先接入：
   - 小龙虾私聊对话。
   - 群聊总结 / @ 信息理解。
   - 工作记录 / 小龙虾第一人称日记生成。
3. 真实 AI 使用 OpenAI 兼容接口，便于适配 DeepSeek、Kimi 2.6 等主流模型。
4. 前端不直接调 AI；真实 AI key、模型、base URL、提示词和请求日志都放在 OpenClaw 后端。
5. 仍需保留 mock fallback：AI 接口失败、超时或未配置时，Demo 继续可演示。
6. 暂不接真实 QQ 登录、真实群消息、真实消息发送。
7. 阶段 1.5 不是一次性完成后就不维护的模块，而是后续功能阶段持续演进的数据与 AI 底座。
8. 后续每个功能阶段都必须同步考虑前端交互、OpenClaw API、SQLite 数据记录、真实 AI adapter、mock fallback 和演示降级。
9. 阶段 2 原本是“小龙虾认养流程”，但该能力已经并入阶段 1 完成；阶段 2 不再单独执行。
10. OpenClaw 的核心结构应为 `Memory + Capabilities + Tools + Review + AI Adapter`。
11. `capabilities` 记录小龙虾能做什么、触发方式、权限要求、风险等级和输出类型。
12. `tools` 记录小龙虾靠什么做，包括输入输出 schema、所需权限、风险等级、是否需要确认、是否有 mock 工具。
13. `review_policies` 记录审查策略，覆盖前置审查和后置审查，例如禁止读取未授权群、禁止自动发送、分享前脱敏。
14. `tool_runs` 和 `review_results` 必须记录工具调用和审查结果，让工作记录、日记和调试都可追溯。
15. 后续流程必须稳定协调：意图/能力识别 -> 权限检查 -> 工具选择 -> 前置审查 -> 工具执行 -> AI 生成 -> 后置审查 -> 预览/确认 -> 事件和记忆写入。
16. 小龙虾聊天框输出应支持流式输出，用来提升真实 AI 体感和陪伴感。
17. 流式输出不能绕过审查：普通闲聊、解释、低风险文本可以边生成边展示；回复草稿、空间动态、分享卡片、日记等需要结构化保存或高风险确认的内容，必须完成后置审查后再解锁操作按钮。
18. 群聊相关能力的差异化表达要从“群消息总结”调整为“主动巡群提醒”：元宝 AI 更像用户打开群后的手动总结引擎，QQClaw 是用户授权给个人宠物 Agent 后，主动巡看不常打开但不想错过的群，只提醒值得关注的内容并引导用户跳回原群。
19. QQClaw 不应被表达为替代元宝 AI；更好的表述是“QQClaw 可调用元宝 AI 的群消息理解能力，但负责跨群授权、主动感知、提醒判断、宠物化表达、工作记录和回到原群”。

## 最新边界修正

1. 认养窗口不能暗示小龙虾已经帮用户捞过 @ 或处理过消息；认养前只能表达“刚出现、尚未使用、等待用户授权”。
2. 未认养前，群聊右侧只能是群资料、成员、文件等 QQ 原生信息，不出现小龙虾状态、阶段任务、打卡或奖励。
3. 认养后进入小龙虾私聊界面。该界面结构为：左侧会话索引、中间小龙虾聊天、右侧小龙虾形象和打卡/奖励。
4. 产品内不使用“阶段任务”称呼；面向用户的引导称为“打卡”，打卡越多，解锁挂饰、徽章、空间装扮等奖励。
5. 阶段 4 中“去空间”和当前打卡按钮仍用于录屏推进引导；“去权限”“生成感知卡”“查看感知卡”“跳回群消息”“查看群摘要”“写回复草稿”“查看工作记录”已接入真实权限、卡片、来源跳转和 OpenClaw 记录闭环。
6. 打卡呈现不应像调试面板。右侧只承载当前下一步、轻量进度和奖励预告；已完成项用点状进度表达，完整历史后续再进工作记录或奖励页。
7. 核心文本卡片不应替代用户阅读原群；默认做发现和提醒，必要时展开简短摘要，并始终保留跳回原群路径。

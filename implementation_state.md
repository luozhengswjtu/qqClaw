# QQ 养虾 v2 实施状态

更新时间：2026-04-30

## 当前结论

项目方向已从 v1 的“QQ 养虾独立产品工作台”调整为 v2 的“QQ 主界面里的小龙虾 Agent 层”。

核心产品体验：

1. 用户像往常一样打开 QQ。
2. 在合适时机看到小龙虾动画入口。
3. 点击后完成小龙虾认养。
4. 进入小龙虾主交互界面。
5. 在引导任务中一步步体验能力：对话、授权、收集 @、总结群聊、回复草稿、工作记录、空间动态、评论回复。
6. 日记作为隐藏惊喜，在用户完成若干操作后触发。
7. 小龙虾空间是小龙虾自己的 QQ 空间，动态由小龙虾发布。

## 硬约束

1. v2 从零实现，不依赖任何历史 Demo 目录、旧实现代码或旧信息架构。
2. 历史实现目录若仍存在，应视为可删除的废弃产物，不作为代码或设计基线。
3. 新实现应另起目录，建议为 `apps/qq-lobster-v2`。
4. 第一屏必须是 QQ 主界面，不是产品首页、功能导航或营销页。
5. 小龙虾首次出现应是 QQ 使用过程中的自然触发，不是用户主动打开功能中心。
6. 日记第一次出现必须是隐藏惊喜，不是用户点击“生成日记”。
7. 工作记录、@ 消息、群聊总结、回复草稿优先以聊天窗口里的文本卡片呈现。
8. 日记、成就、分享、空间动态优先以强视觉卡片呈现。
9. 所有 AI 输出必须通过 adapter 层，禁止直接散落在页面 JSX 中。

## 用户偏好

1. 用户希望分阶段实现，每阶段完成后由用户体验并反馈。
2. 用户愿意动态调整方向。
3. 用户不希望一次性让 AI 完整生成 Demo。
4. 用户重视 QQ 生态中的自然出现感。
5. 用户重视小龙虾情绪价值和惊喜感。
6. 用户重视后续可接真实 AI，因此底层框架要稳定。
7. 用户可以使用 `gpt-image-2` 协助生成图片资产，后续需要把形象、动画关键帧、日记卡、空间头图、成就挂饰等资产需求及时拆给用户。
8. 小龙虾形象风格应偏 Q 版可爱，提高用户接触意愿；不要做人形拟人化，重点是“可爱小动物伙伴”，不是“人形 AI 助手”。
9. 第一阶段 Demo 范围已对齐：先做 QQ 主界面、小龙虾首次自然出现、认养入口和认养流程，不急着做完整功能闭环。
10. 讨论中形成的新信息要及时写入文档，方便新线程快速接手。

## 生图协作记录

用户可用 `gpt-image-2` 生成视觉资产。当前建议优先准备：

1. Q 版小龙虾主形象，非拟人，透明背景。
2. 首次出现动画关键帧：气泡冒出、小钳子伸出、探头、眨眼、弹出提示。
3. 小龙虾状态图：开心、认真工作、得意、写日记、委屈或出错。
4. 日记惊喜卡视觉：适合在 QQ 内弹出的手账感/空间动态感卡片。
5. 成就和挂饰素材：第一捞 @、群聊总结、回复草稿、第一篇日记、空间营业中。
6. 小龙虾空间头图：QQ 空间感、小龙虾自己的主页氛围。

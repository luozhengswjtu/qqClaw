# QQClaw 兴趣爱好系统讨论总结

更新时间：2026-05-02

## 1. 讨论背景

最初的 QQClaw 认养流程里包含“兴趣爱好”属性，但如果兴趣只作为资料标签存在，价值很弱。经过讨论，我们认为兴趣爱好应该成为 QQClaw 长期陪伴、主动服务、社交传播和 OpenClaw skill 系统的重要入口。

核心判断：

> 兴趣爱好不是静态标签，而是 QQClaw 持续理解用户、主动提醒用户、陪用户表达和帮用户发现同好的生活线索。

这条方向会强化 QQClaw 的两种属性：

1. 助手属性：围绕用户兴趣主动看信息、做摘要、提醒关键内容。
2. 宠物属性：小龙虾也会被用户“养成”某种兴趣人格，像真实同好一样表达和互动。

## 2. 兴趣系统的核心升级

原始设计：

```text
用户选择兴趣 -> 小龙虾资料页显示兴趣标签
```

升级后：

```text
用户选择兴趣
  -> 安装或启用相关 skills
  -> 配置关注来源、频率、城市、关键词和提醒边界
  -> OpenClaw 定期巡看相关内容
  -> QQClaw 用同好语气提醒用户
  -> 生成日记素材、空间动态、成就、挂饰和社交机会
```

因此兴趣爱好应承担以下功能：

1. 内容订阅来源。
2. 主动提醒来源。
3. 小龙虾人格塑造来源。
4. 日记和空间动态素材来源。
5. 成就、挂饰、装扮的解锁来源。
6. 同好社交机会的发现来源。
7. 腾讯生态数据接入的授权入口。

## 3. 兴趣系统的三层结构

### 3.1 用户兴趣画像

记录用户喜欢什么，示例：

1. 音乐。
2. 二次元。
3. 羽毛球。
4. 游戏。
5. AI 工具。
6. 摄影。
7. 校园活动。
8. 演唱会 / 展会 / 赛事。

这层回答：

> 用户关心什么？

### 3.2 小龙虾兴趣人格

兴趣不只是“用户喜欢”，还应影响小龙虾自身的表达方式。

目标是让用户感觉：

> 这只小龙虾是被我养出来的，它也逐渐变成了我的同好。

示例：

羽毛球兴趣：

```text
队长，我今天刷到一个双打轮转小技巧，感觉很适合你上次说的那个站位问题。
我先夹在钳子里，晚上你要不要试试？
```

二次元兴趣：

```text
队长，本周你所在城市好像有个同人展。
我把时间和地点捞出来了，虽然我不能去，但我可以负责帮你看摊位图。
```

音乐兴趣：

```text
队长，你最近循环的那个歌手好像有新动态。
我听不懂高音，但我知道这条你可能会想看。
```

这层回答：

> 小龙虾如何像真实爱好者一样陪用户聊兴趣？

### 3.3 OpenClaw Skill / 兴趣连接器

兴趣需要落到可运行能力上，而不是只停留在文案上。

一个兴趣可以对应多个 skill：

```text
interest: 羽毛球
skills:
- badminton_news_digest
- match_schedule_watch
- local_court_event_watch
- training_tip_daily
- lobster_space_sport_post
```

```text
interest: 音乐
skills:
- music_daily_digest
- artist_event_watch
- song_chat
- playlist_recommend
- concert_reminder
- lobster_space_music_post
```

这层回答：

> OpenClaw 具体靠什么能力持续服务用户？

## 4. 兴趣 Skill 的能力分类

我们讨论出一个有用的抽象：兴趣 skill 可以分成“看、想、说、找”四类。

### 4.1 看

QQClaw 定期关注用户配置的来源。

示例：

1. 每天看指定网站。
2. 关注比赛赛程。
3. 关注歌手动态。
4. 关注本地活动。
5. 关注展会、同人展、音乐会。
6. 关注用户授权的腾讯业务数据。

### 4.2 想

OpenClaw 判断哪些内容值得提醒。

判断依据：

1. 是否和用户兴趣高度相关。
2. 是否和用户所在城市相关。
3. 是否有截止时间、报名时间、演出时间。
4. 是否来自用户明确关注的人、团队、歌手、赛事。
5. 是否符合用户设置的提醒频率。

### 4.3 说

QQClaw 用真实同好语气表达，而不是像新闻机器人。

表达应该同时具备：

1. 助手感：信息清楚、来源明确。
2. 宠物感：语气有小龙虾人格。
3. 同好感：说法像真的懂这个兴趣。

### 4.4 找

QQClaw 主动发现社交机会。

示例：

1. 推荐相关 QQ 群。
2. 推荐同城活动。
3. 推荐校园社团。
4. 推荐兴趣社区。
5. 推荐比赛、展会、球局、音乐会。

边界：

1. 只能推荐，不能自动加群。
2. 不能自动私聊群主。
3. 不能自动替用户报名。
4. 用户决定是否进入原场景。

## 5. 和腾讯生态的结合

这一点是兴趣系统的重要亮点。QQClaw 不是泛用 AI 助手，它可以成为腾讯生态里连接用户兴趣生活的宠物 Agent。

### 5.1 QQ 音乐

可授权数据：

1. 常听歌手。
2. 最近循环歌曲。
3. 收藏歌曲。
4. 收藏歌单。
5. 曲风偏好。
6. 关注歌手。
7. 演唱会 / 新歌 / 专辑动态。

可提供能力：

1. 新歌提醒。
2. 演唱会提醒。
3. 歌单推荐。
4. 歌曲讨论。
5. 音乐兴趣日报。
6. 小龙虾空间音乐动态。

示例链路：

```text
用户授权 QQ 音乐
  -> OpenClaw 读取音乐画像
  -> 写入 memories：用户喜欢某歌手、常听粤语歌
  -> QQClaw 关注歌手演出动态
  -> 发现演唱会信息
  -> 生成提醒卡
  -> 小龙虾用同好语气提醒
  -> 生成空间动态预览
  -> 用户确认后发布
```

### 5.2 腾讯视频

可授权数据：

1. 追剧记录。
2. 综艺偏好。
3. 动漫偏好。
4. 收藏内容。
5. 更新提醒。

可提供能力：

1. 追剧更新提醒。
2. 剧情回顾。
3. 追番日历。
4. 同类内容推荐。
5. 空间动态：小龙虾追番值班。

### 5.3 腾讯游戏 / WeGame

可授权数据：

1. 常玩游戏。
2. 战绩 / 段位。
3. 活动信息。
4. 赛事关注。

可提供能力：

1. 版本更新摘要。
2. 赛事提醒。
3. 好友开黑提醒。
4. 活动奖励提醒。
5. 空间动态：陪队长上分。

### 5.4 腾讯文档 / 会议 / 邮箱

适合偏学习、项目、工作场景。

可提供能力：

1. DDL 提醒。
2. 项目资料整理。
3. 群聊任务和文档联动。
4. 工作记录沉淀。

这类内容更适合私聊提醒和工作记录，不一定适合公开空间动态。

### 5.5 腾讯地图 / 本地生活

可授权数据：

1. 用户所在城市。
2. 常用区域。
3. 本地活动偏好。

可提供能力：

1. 本地漫展提醒。
2. 演唱会提醒。
3. 球馆活动提醒。
4. 展览、赛事、校园活动提醒。

隐私要求：

1. 城市信息必须用户授权。
2. 不应暴露详细住址。
3. 空间动态发布前必须脱敏。

### 5.6 QQ 群 / QQ 频道 / 兴趣社区

这是最贴 QQClaw 的社交方向。

可提供能力：

1. 发现相关兴趣群。
2. 推荐同好社区。
3. 提醒用户某个兴趣群出现重要活动。
4. 汇总同好讨论热点。

边界：

1. 只能推荐，不自动加入。
2. 只展示申请入口和注意事项。
3. 用户自己决定是否加入。

## 6. 兴趣连接器概念

为了把手动兴趣和腾讯生态数据统一起来，可以引入 `interest_connectors`。

示例：

```text
connector: qq_music
interest: music
data_scope:
- top_artists
- liked_songs
- playlists
- recent_play
- concert_watch

skills:
- music_daily_digest
- artist_event_watch
- song_chat
- playlist_recommend
- lobster_space_music_post
```

通用结构：

```text
Interest
  -> Connector
  -> Skill
  -> Tool
  -> Review
  -> Memory
  -> Output
```

兴趣连接器的价值：

1. 把“我喜欢音乐”变成更细的用户记忆。
2. 把腾讯生态数据转化为 QQClaw 的主动服务能力。
3. 让 QQClaw 的推荐和提醒更可信。
4. 让用户感觉“小龙虾真的懂我”。
5. 解释为什么这个产品适合腾讯做，而不是任何通用 AI 助手都能做。

## 7. 和 OpenClaw Agent 底座的关系

兴趣系统必须接入 OpenClaw 的现有底座：

```text
Memory + Capabilities + Tools + Review + AI Adapter
```

### 7.1 Memory

兴趣相关记忆分层：

1. 身份记忆：用户选择了哪些兴趣。
2. 权限记忆：用户授权了哪些来源和连接器。
3. 行为记忆：QQClaw 做过哪些兴趣巡看、提醒、推荐。
4. 能力记忆：QQClaw 当前安装了哪些兴趣 skills。
5. 语义记忆：用户长期喜欢什么风格、歌手、队伍、活动、表达方式。

### 7.2 Capabilities

兴趣能力需要注册，例如：

```text
interest_daily_digest
artist_event_watch
local_acg_event_watch
badminton_match_watch
interest_community_discovery
lobster_space_interest_post
```

每个能力需要说明：

1. 触发方式。
2. 所需权限。
3. 风险等级。
4. 输出类型。
5. 是否需要用户确认。

### 7.3 Tools

兴趣工具示例：

```text
fetch_interest_sources
read_qq_music_profile
watch_artist_events
watch_local_events
discover_interest_groups
generate_interest_digest
generate_interest_space_post
```

第一版可以全部 mock，后续再逐步接真实接口。

### 7.4 Review

兴趣系统必须走审查：

1. 外部内容需要来源标记。
2. 活动和票务信息需要提示可能变化。
3. 城市信息必须脱敏。
4. 空间动态发布前必须预览确认。
5. 推荐 QQ 群不能自动加入。
6. 高风险建议不得直接下结论，例如医疗、金融、法律、极端运动等。

### 7.5 AI Adapter

真实 AI 负责：

1. 把兴趣内容转成小龙虾语气。
2. 生成简报。
3. 生成空间动态草稿。
4. 生成日记素材。
5. 和用户进行兴趣对话。

但 AI 不能绕过：

1. 权限。
2. 工具注册。
3. 审查策略。
4. 用户确认。

## 8. 提醒频率和用户控制

兴趣系统需要“主动但不烦”。

可配置项：

1. 每天几点提醒。
2. 每周几次提醒。
3. 只提醒重点事件。
4. 只提醒本地活动。
5. 只提醒关注歌手 / 队伍 / 关键词。
6. 是否写入日记素材。
7. 是否允许生成小龙虾空间动态预览。
8. 是否允许推荐 QQ 群或社区。

提醒等级：

1. 轻提醒：每日兴趣简报。
2. 重点提醒：演唱会、报名截止、比赛时间、活动开票。
3. 惊喜提醒：高度匹配的内容。
4. 安静模式：不主动打扰，只写入日记或等待用户询问。

## 9. 小龙虾空间和兴趣表达

兴趣内容很适合进入小龙虾空间，但必须保持宠物表达。

羽毛球空间动态示例：

```text
今天替队长巡了一圈羽毛球消息。
发现一个双打轮转技巧，我先偷偷练了三遍，虽然我只有钳子。
```

二次元空间动态示例：

```text
今天的巡展任务完成。
我发现队长所在城市下周有个同人展，地图我已经夹在钳子里了。
```

音乐空间动态示例：

```text
今天陪队长巡音乐宇宙。
发现他最近循环的歌手有演出消息，我已经把小耳机挂好了。
```

空间动态规则：

1. 动态由小龙虾发布。
2. 人类用户确认后发布。
3. 不暴露隐私。
4. 不直接搬运外部内容。
5. 用小龙虾自身视角表达。

## 10. 兴趣驱动的成长和装扮

兴趣系统可以强化宠物养成。

示例：

| 兴趣 | 行为 | 奖励 |
| --- | --- | --- |
| 羽毛球 | 连续 7 天巡看赛事 / 技巧 | 小球拍挂饰 |
| 二次元 | 巡展 3 次 | 应援棒 |
| 音乐 | 生成 5 次音乐简报 | 小耳机 / 唱片挂饰 |
| AI 工具 | 摘要 5 次工具动态 | 赛博眼镜 |
| 摄影 | 记录 3 次本地展览 | 小相机 |
| 游戏 | 关注 3 次赛事 / 版本更新 | 手柄挂饰 |

这种设计让兴趣不只是信息服务，也会反哺：

1. 成就。
2. 挂饰。
3. 小龙虾形象变化。
4. 空间动态。
5. 长期留存。

## 11. 同好社交机会

兴趣系统还可以帮助用户发现新的 QQ 社交关系。

示例提醒：

```text
我发现 3 个可能适合你的羽毛球 QQ 群：
1. 校内羽毛球约球群
2. 周末双打搭子群
3. 同城球馆活动群

要不要我帮你整理一下入群前需要看的信息？
```

原则：

1. 只推荐，不自动加入。
2. 用户决定是否申请。
3. 推荐理由要清楚。
4. 如果来自外部平台或公开社区，要标明来源。
5. 不制造社交压力。

这条能力非常贴合 QQ，因为 QQ 的优势就是群、频道、兴趣社区和长期关系。

## 12. 和 3 分钟 Demo 的关系

我们讨论过：3 分钟 Demo 内容不能太多，需要做减法。

兴趣系统可以作为 Demo 的亮点，但不应把所有兴趣能力都展示出来。

推荐 Demo 主线：

```text
打开 QQ
  -> 小龙虾自然出现
  -> 认养
  -> 选择音乐兴趣
  -> 模拟授权 QQ 音乐
  -> 小龙虾发现歌手演唱会 / 新歌动态
  -> 用同好语气提醒
  -> 生成小龙虾空间动态预览
  -> 用户确认发布到小龙虾空间
  -> 好友看到后产生兴趣
```

群聊巡群能力作为基础能力短展示，不做主角：

```text
除了兴趣陪伴，小钳也能在授权后帮你巡群，提醒不常打开但不能错过的群。
```

3 分钟内应突出三个点：

1. QQ 原生感：不是独立 App。
2. 兴趣连接器：接腾讯生态数据，让小龙虾更懂用户。
3. 宠物式社交表达：小龙虾用自己的语气提醒、写动态、发空间。

不在 3 分钟里展开：

1. 完整工具注册表。
2. 完整审查流程。
3. 多兴趣配置。
4. 多平台连接器。
5. 完整社区推荐流程。
6. 复杂成就系统。

这些放到说明文档或答辩 Q&A。

## 13. 和小龙虾 / QQ 宠物形象的关系

关于是否直接使用 QQ 宠物形象，我们讨论后暂不改方向。

当前判断：

1. 不建议完全放弃小龙虾形象。
2. 小龙虾仍然贴合“QQ 养虾”赛题。
3. 应把小龙虾做成“QQ 宠物体系里的新宠物形态”。
4. 继承 QQ 宠物的认养、陪伴、成长、装扮、心情、主动找你等关系模型。
5. 视觉上可以 QQ 宠物化：Q 版、圆润、胖嘟嘟、有表情、简单服饰和挂饰。

一句话：

> QQClaw 是 QQ 宠物在 AI 时代的新形态：一只会巡群、懂兴趣、会写日记、会发空间动态，也会帮你连接同好的小龙虾。

## 14. 产品价值总结

兴趣爱好系统让 QQClaw 从“群聊 AI 助手”升级为“懂我兴趣的 QQ 宠物伙伴”。

它带来的价值：

1. 助手价值：帮用户看信息、总结、提醒。
2. 陪伴价值：像同好一样和用户聊兴趣。
3. 宠物价值：小龙虾被用户养出兴趣人格。
4. 社交价值：发现 QQ 群、社区、同城活动和同好机会。
5. 生态价值：连接 QQ 音乐、腾讯视频、游戏、地图、频道、群等腾讯业务。
6. 长期留存：兴趣触发每日简报、空间动态、成就和挂饰。
7. 差异化：这不是通用 AI 助手，而是腾讯生态里的 AI 宠物 Agent。

## 15. 后续可继续讨论的问题

1. 初赛 Demo 是否把“音乐兴趣 + QQ 音乐模拟授权”作为主亮点？
2. 第一版兴趣 skill 只做一个兴趣，还是保留多个兴趣入口但只演示一个？
3. 兴趣提醒频率默认如何设置才不打扰？
4. 小龙虾空间动态是否默认只生成预览，不自动发布？
5. 兴趣连接器是否需要在 UI 中显式命名？
6. 社区推荐是否进入初赛 Demo，还是放在说明文档中作为扩展能力？
7. 兴趣记忆是否需要可查看、可删除、可关闭？
8. 如何让“真实爱好者语气”既有共鸣，又不过度装熟？
